# 1단계 — 내 PC에 개발 환경 만들기

> 목표: 조제님의 Windows PC에 새 도서관 시스템을 만들 "작업실"을 차립니다.
> 설치할 것은 4가지 도구이고, 마지막에 GitHub과 연결까지 합니다.
> 소요 시간: 약 1시간 (다운로드 속도에 따라 다름)

---

## 시작하기 전에 — 오늘 설치할 4가지 도구

| 도구 | 하는 일 | 비유하자면 |
|------|---------|-----------|
| **VS Code** | 코드를 쓰는 편집기 | 공책과 연필 |
| **Node.js** | 자바스크립트 코드를 실행하는 엔진 | PHP를 실행해 주던 Apache의 PHP 모듈 같은 존재 |
| **Git** | 코드의 변경 역사를 기록하는 도구 | 게임의 "세이브 파일" 기능 |
| **PostgreSQL** | 데이터베이스 | 책·회원·대출 기록을 담을 서류 캐비닛 |

각 단계 끝에 ✅ **확인하기**가 있습니다. 확인이 성공해야 다음 단계로 넘어가세요.

---

## 1. VS Code 설치 (약 10분)

1. 인터넷 브라우저에서 https://code.visualstudio.com 에 접속합니다.
2. 파란색 **Download for Windows** 버튼을 눌러 설치 파일을 받습니다.
3. 받은 파일을 실행하고, 계속 **다음**을 누릅니다.
   - 중간에 "추가 작업 선택" 화면이 나오면 **"PATH에 추가"** 와 **"Code(으)로 열기"** 관련 체크박스는 모두 켜 두는 것을 추천합니다.
4. 설치가 끝나면 VS Code를 실행해 봅니다.

한글 메뉴를 원하시면: VS Code 왼쪽 맨 아래 네모 4개 아이콘(확장, Extensions)을 누르고 `Korean`을 검색해서 **Korean Language Pack**을 설치한 뒤 재시작하면 메뉴가 한국어로 바뀝니다.

✅ **확인하기**: VS Code가 실행되면 성공.

---

## 2. Node.js 설치 (약 10분)

1. https://nodejs.org 에 접속합니다.
2. **LTS**라고 적힌 버전을 다운로드합니다.
   - LTS는 "Long Term Support(오래 지원되는 안정 버전)"라는 뜻입니다. 숫자가 더 큰 다른 버전이 보여도 LTS를 고르세요.
3. 받은 파일을 실행하고 계속 **Next**를 누릅니다. 특별히 바꿀 설정은 없습니다.
4. 설치가 끝나면 **PC를 다시 시작하거나, 열려 있던 터미널 창을 모두 닫았다가 새로 여세요.** (설치 직후에는 기존 창이 새 프로그램을 못 알아봅니다.)

### 터미널 여는 법 (앞으로 계속 쓰게 됩니다)

- 키보드에서 `윈도우 키`를 누르고 `terminal` 또는 `터미널`을 입력한 뒤 **터미널(Terminal)** 앱을 실행합니다.
- 까만(또는 파란) 창에 글자를 입력하는 화면이 나오면 그게 터미널입니다. 명령어를 입력하고 `Enter`를 치면 실행됩니다.

✅ **확인하기**: 터미널에 아래 두 줄을 한 줄씩 입력하고 Enter를 칩니다.

```
node -v
npm -v
```

각각 `v22.14.0`, `10.9.2` 같은 **버전 숫자**가 나오면 성공입니다. (숫자는 달라도 됩니다.)
"찾을 수 없습니다"라고 나오면 → 터미널을 완전히 닫고 새로 연 뒤 다시 해 보세요. 그래도 안 되면 PC 재시작.

---

## 3. Git 설치 (약 10분)

1. https://git-scm.com 에 접속합니다.
2. **Download for Windows**를 눌러 설치 파일을 받습니다.
3. 설치 화면에서 물어보는 것이 많은데, **전부 기본값 그대로 Next**를 눌러도 됩니다.
4. 설치가 끝나면 터미널을 새로 열고, 아래처럼 **내 이름과 이메일을 Git에 알려줍니다.**
   (코드 변경 기록에 "누가 저장했는지" 적히는 이름표입니다. 이메일은 GitHub 가입에 쓸 이메일과 같게 하세요.)

```
git config --global user.name "Josee"
git config --global user.email "gravityofu@gmail.com"
```

✅ **확인하기**:

```
git --version
```

`git version 2.x.x` 같은 버전이 나오면 성공.

---

## 4. PostgreSQL 설치 (약 15분)

1. https://www.postgresql.org/download/windows/ 에 접속합니다.
2. **Download the installer** 링크를 눌러 EDB 설치 파일 페이지로 이동한 뒤, 최신 안정 버전의 **Windows x86-64** 설치 파일을 받습니다.
3. 설치 파일을 실행하고 **Next**를 누르다 보면 아래 두 화면에서만 멈춰서 확인하세요.

   **① 비밀번호 설정 화면 (가장 중요!)**
   - 데이터베이스 최고 관리자(`postgres` 계정)의 비밀번호를 정하는 화면입니다.
   - 비밀번호를 정하고 **반드시 종이나 메모장에 적어 두세요.** 잊어버리면 꽤 번거로워집니다.

   **② 포트(Port) 화면**
   - `5432`라고 적혀 있으면 그대로 둡니다. (데이터베이스가 사용하는 "문 번호"입니다.)

4. 마지막에 "Stack Builder를 실행할까요?"라고 물으면 **체크를 해제**하고 끝냅니다. (지금은 필요 없습니다.)

설치하면 **pgAdmin**이라는 프로그램도 같이 설치됩니다. 데이터베이스 안을 눈으로 볼 수 있는 관리 도구인데, 다음 단계에서 사용법을 다룰 예정이니 지금은 몰라도 됩니다.

✅ **확인하기**: 터미널을 **새로** 열고 아래를 입력합니다.

```
psql -U postgres
```

비밀번호를 물어보면 아까 정한 비밀번호를 입력합니다. (입력해도 화면에 아무것도 안 보이는 게 정상입니다. 그냥 입력하고 Enter.)
`postgres=#` 라는 표시가 나오면 성공! 데이터베이스 안에 들어온 것입니다. `\q` 를 입력하고 Enter를 치면 밖으로 나옵니다.

> `psql`을 찾을 수 없다고 나오면: PostgreSQL이 PATH에 등록되지 않은 경우입니다. 이 확인은 pgAdmin 실행으로 대신해도 됩니다(시작 메뉴에서 pgAdmin 4 실행 → 왼쪽 Servers 클릭 → 비밀번호 입력 → 연결되면 성공).

## 5. GitHub 계정과 저장소 만들기 (약 10분)

GitHub은 내 코드를 인터넷 금고에 보관해 주는 서비스입니다. 이걸 해 두면 다른 PC나 노트북에서도 이어서 작업할 수 있습니다.

1. https://github.com 에 접속해서 **Sign up**으로 계정을 만듭니다. (이미 있다면 로그인)
   - 이메일은 3번 단계에서 Git에 등록한 것과 같은 이메일을 쓰세요.
2. 로그인 후 오른쪽 위 **+** 버튼 → **New repository**를 누릅니다.
3. 아래처럼 입력합니다.
   - **Repository name**: `LibraryNearMe`
   - **Public / Private**: **Private** 추천 (비공개. 나중에 언제든 공개로 바꿀 수 있습니다)
   - **Add a README file**: ✔ 체크
   - **Add .gitignore**: 목록에서 **Node** 선택 (Git이 기록하면 안 되는 파일들을 자동으로 걸러 주는 설정입니다)
4. 초록색 **Create repository** 버튼을 누릅니다.

✅ **확인하기**: `github.com/내아이디/LibraryNearMe` 페이지가 열리고 README 파일이 보이면 성공.

## 6. 내 PC 폴더와 GitHub 연결하기 (약 10분)

이제 인터넷 금고(GitHub)에 있는 저장소를 내 PC의 `C:\projects\LibraryNearMe` 폴더로 가져옵니다.

1. GitHub의 LibraryNearMe 페이지에서 초록색 **<> Code** 버튼을 누르고, **HTTPS** 주소를 복사합니다.
   (`https://github.com/내아이디/LibraryNearMe.git` 같은 모양입니다.)
2. 터미널을 열고 아래를 한 줄씩 입력합니다. (`주소` 자리에 방금 복사한 주소를 붙여넣기: 터미널에서는 `Ctrl+V` 또는 마우스 오른쪽 클릭)

```
cd C:\projects\LibraryNearMe
git clone 주소 .
```

   - 마지막의 **점(.)을 빼먹지 마세요!** "지금 이 폴더 안에 받아라"는 뜻입니다.
   - 처음 GitHub과 통신할 때 **브라우저 창이 뜨면서 로그인하라고** 합니다. 로그인하고 허용(Authorize)을 누르면 됩니다. 이건 처음 한 번만 합니다.

3. 잘 연결되었는지 시험 삼아 하나 바꿔서 올려 봅니다. VS Code에서 `C:\projects\LibraryNearMe` 폴더를 열고(메뉴: 파일 → 폴더 열기), `README.md` 파일에 아무 문장이나 한 줄 추가하고 저장한 뒤, 터미널에서:

```
cd C:\projects\LibraryNearMe
git add .
git commit -m "첫 커밋 연습"
git push
```

   이 세 줄의 뜻: `add` = 바뀐 파일을 상자에 담기, `commit` = 상자에 메모를 붙여 봉인(세이브), `push` = 봉인한 상자를 GitHub 금고로 보내기.

✅ **확인하기**: 브라우저에서 GitHub의 LibraryNearMe 페이지를 새로고침했을 때, 방금 추가한 문장이 README에 보이면 **1단계 전체 완료입니다!** 🎉
