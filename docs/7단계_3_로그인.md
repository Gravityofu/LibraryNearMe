# 7단계 (3) — 로그인 기능 만들기

> 목표: 가입한 회원이 아이디·비밀번호로 로그인하고, "출입증(토큰)"을 받아
> 로그인 상태를 유지하게 만듭니다.
> 소요 시간: 약 45분

---

## 오늘의 위치

7단계(계정 시스템) 네 부분 중 **세 번째**예요.

1. 사용자 표 만들기 ✅
2. 회원가입 기능 ✅
3. **로그인 기능 ← 오늘**
4. 관리자 페이지 자물쇠

## 오늘의 핵심 개념: "출입증(토큰)"

로그인을 이렇게 상상해 보세요. 놀이공원 입구에서 표를 보여주면(아이디·비밀번호 확인), 손목에 **밴드**를 채워주죠(토큰 발급). 그 뒤로는 놀이기구를 탈 때마다 표를 다시 살 필요 없이 손목밴드만 보여주면 됩니다.

로그인도 똑같아요. 처음 한 번 아이디·비밀번호를 확인하고 맞으면, 서버가 **토큰**이라는 손목밴드를 줍니다. 그 뒤로는 회원이 무언가 할 때마다 이 토큰만 보여주면 "아, 로그인된 사람이구나" 하고 알아봐요. 이 토큰은 위조할 수 없게 서버가 특별한 비밀 열쇠로 서명해서 만듭니다. 이 기술을 JWT라고 불러요.

---

## 준비

주방·홀 두 서버를 켜두세요 (backend `npm run start:dev`, frontend `npm run dev`).

# A. 주방(NestJS) 쪽 작업

## A-1. 도구 두 개 설치 — 약 3분

주방 서버가 도는 터미널 말고 다른 터미널에서 backend 폴더로 가 설치합니다.

```
cd C:\projects\LibraryNearMe\backend
npm install @nestjs/jwt dotenv
```

- `@nestjs/jwt`: 토큰(손목밴드)을 만들고 검사하는 도구
- `dotenv`: `.env` 파일의 비밀 값을 프로그램이 확실히 읽어오게 도와주는 도구

## A-2. 토큰 서명용 비밀 열쇠 정하기 — 약 3분

토큰을 위조 못 하게 서명하려면 **아무도 모르는 비밀 열쇠**가 필요해요. 이건 `.env` 파일에 적습니다.

`backend` → **`.env`** 파일을 열고, 맨 아래에 한 줄을 추가하세요. 따옴표 안은 **길고 아무렇게나** 적으면 됩니다(키보드를 마구 눌러도 좋아요).

```
JWT_SECRET="k3j2h5g8f0a1s9d7f6g4h2j0k8l6-mysecret-library-2026-x9c2"
```

저장하세요. 이 값은 비밀번호처럼 중요한 거라 `.env`에 두고 GitHub에는 올리지 않습니다(이미 그렇게 설정돼 있어요).

## A-3. .env를 확실히 읽도록 설정 — 약 2분

`backend` → `src` → **`main.ts`** 를 엽니다. 맨 **첫 줄**에 아래 한 줄을 추가하세요. (기존 `import` 들보다 위에 있어야 합니다.)

```typescript
import 'dotenv/config';
```

전체 모양은 이렇게 됩니다.

```typescript
import 'dotenv/config';
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  app.enableCors();
  await app.listen(process.env.PORT ?? 3001);
}
bootstrap();
```

이 한 줄이 "프로그램을 켤 때 `.env`의 비밀 값들을 먼저 불러와"라는 뜻이에요. 방금 만든 `JWT_SECRET`을 확실히 읽게 해줍니다.

## A-4. 로그인 처리 일꾼 만들기 — 약 10분

`backend` → `src` 안에 **`auth`** 폴더를 새로 만들고, 그 안에 **`auth.service.ts`** 파일을 만들어 붙여넣습니다.

```typescript
import { Injectable, UnauthorizedException } from '@nestjs/common';
import { PrismaService } from '../prisma.service';
import { JwtService } from '@nestjs/jwt';
import * as bcrypt from 'bcryptjs';

@Injectable()
export class AuthService {
  constructor(
    private prisma: PrismaService,
    private jwt: JwtService,
  ) {}

  async login(data: { loginId: string; password: string }) {
    // 1. 어느 도서관인지 (지금은 첫 번째 도서관)
    const library = await this.prisma.library.findFirst();

    // 2. 아이디로 회원 찾기
    const user = library
      ? await this.prisma.user.findFirst({
          where: { libraryId: library.id, loginId: data.loginId },
        })
      : null;

    // 회원이 없거나 비밀번호가 없으면 로그인 실패
    if (!user || !user.passwordHash) {
      throw new UnauthorizedException(
        '아이디 또는 비밀번호가 올바르지 않습니다.',
      );
    }

    // 3. 입력한 비밀번호가 저장된 암호와 맞는지 확인
    const ok = await bcrypt.compare(data.password, user.passwordHash);
    if (!ok) {
      throw new UnauthorizedException(
        '아이디 또는 비밀번호가 올바르지 않습니다.',
      );
    }

    // 4. 맞으면 "출입증(토큰)" 발급 (누구인지·역할·소속도서관을 담아 서명)
    const token = await this.jwt.signAsync({
      sub: user.id,
      role: user.role,
      libraryId: user.libraryId,
    });

    // 5. 토큰과 기본 정보를 돌려줍니다.
    return {
      token,
      user: { id: user.id, name: user.name, role: user.role },
    };
  }
}
```

핵심은 `bcrypt.compare(입력한 비번, 저장된 암호)` 예요. 암호화된 비밀번호는 되돌릴 수 없다고 했죠? 그래서 "복호화해서 비교"하는 게 아니라, **입력한 비번을 똑같이 암호화해서 맞는지 대조**합니다. 그리고 한 가지 안전 요령 — 아이디가 없든 비밀번호가 틀리든 **똑같은 메시지**를 보여줘요. "그 아이디는 있는데 비번만 틀렸다" 같은 힌트를 안 주기 위해서입니다.

## A-5. 로그인 요청을 받는 창구 만들기 — 약 3분

같은 `auth` 폴더에 **`auth.controller.ts`** 파일을 만들어 붙여넣습니다.

```typescript
import { Controller, Post, Body } from '@nestjs/common';
import { AuthService } from './auth.service';

@Controller('auth')
export class AuthController {
  constructor(private authService: AuthService) {}

  // POST /auth/login : 로그인
  @Post('login')
  login(@Body() body: { loginId: string; password: string }) {
    return this.authService.login(body);
  }
}
```

## A-6. 주방에 등록하기 — 약 4분

`backend` → `src` → **`app.module.ts`** 를 열고 아래로 통째로 바꿔주세요. (토큰 도구 `JwtModule`과 `Auth` 부품들이 추가됐어요.)

```typescript
import { Module } from '@nestjs/common';
import { JwtModule } from '@nestjs/jwt';
import { AppController } from './app.controller';
import { AppService } from './app.service';
import { PrismaService } from './prisma.service';
import { LibraryController } from './library/library.controller';
import { LibraryService } from './library/library.service';
import { UsersController } from './users/users.controller';
import { UsersService } from './users/users.service';
import { AuthController } from './auth/auth.controller';
import { AuthService } from './auth/auth.service';

@Module({
  imports: [
    JwtModule.register({
      global: true,
      secret: process.env.JWT_SECRET,
      signOptions: { expiresIn: '7d' }, // 토큰은 7일간 유효
    }),
  ],
  controllers: [
    AppController,
    LibraryController,
    UsersController,
    AuthController,
  ],
  providers: [
    AppService,
    PrismaService,
    LibraryService,
    UsersService,
    AuthService,
  ],
})
export class AppModule {}
```

저장 후 주방 서버가 다시 시작되며 `successfully started`가 나오는지 확인하세요.

> 만약 서버가 다시 안 켜지거나 에러가 나면, `.env`에 `JWT_SECRET`을 저장했는지, `main.ts` 첫 줄에 `import 'dotenv/config';`가 있는지 다시 확인하세요.

# B. 홀(Next.js) 쪽 작업 — 로그인 화면

## B-1. /login 페이지 만들기 — 약 12분

`frontend` → `src` → `app` 안에 **`login`** 폴더를 만들고, 그 안에 **`page.tsx`** 파일을 만들어 붙여넣습니다.

```tsx
"use client";

import { useState } from "react";
import { Button } from "@/components/ui/button";
import { Input } from "@/components/ui/input";
import { Label } from "@/components/ui/label";
import { Card, CardContent, CardHeader, CardTitle } from "@/components/ui/card";

const API_URL = "http://localhost:3001";

export default function LoginPage() {
  const [loginId, setLoginId] = useState("");
  const [password, setPassword] = useState("");
  const [status, setStatus] = useState("");

  async function handleLogin() {
    setStatus("로그인 중...");
    const res = await fetch(`${API_URL}/auth/login`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ loginId, password }),
    });
    if (res.ok) {
      const data = await res.json();
      // 받은 출입증(토큰)을 브라우저에 보관 → 로그인 상태 유지
      localStorage.setItem("token", data.token);
      localStorage.setItem("userName", data.user.name);
      setStatus(`✅ ${data.user.name}님, 환영합니다!`);
    } else {
      const data = await res.json().catch(() => null);
      setStatus("❌ " + (data?.message || "로그인에 실패했습니다."));
    }
  }

  return (
    <main className="mx-auto max-w-md p-8">
      <Card>
        <CardHeader>
          <CardTitle>로그인</CardTitle>
        </CardHeader>
        <CardContent className="flex flex-col gap-4">
          <div className="flex flex-col gap-2">
            <Label htmlFor="loginId">아이디</Label>
            <Input id="loginId" value={loginId} onChange={(e) => setLoginId(e.target.value)} />
          </div>
          <div className="flex flex-col gap-2">
            <Label htmlFor="password">비밀번호</Label>
            <Input id="password" type="password" value={password} onChange={(e) => setPassword(e.target.value)} />
          </div>
          <Button onClick={handleLogin}>로그인</Button>
          <p className="text-sm text-muted-foreground">{status}</p>
        </CardContent>
      </Card>
    </main>
  );
}
```

`localStorage.setItem("token", ...)` 이 부분이 받은 손목밴드(토큰)를 브라우저 서랍에 넣어두는 거예요. 다음에 로그인이 필요한 화면에서 이 토큰을 꺼내 씁니다.

## C. 확인하기 — 약 5분

1. 브라우저에서 `http://localhost:3000/login` 접속
2. 7단계(2)에서 **가입할 때 정한 아이디·비밀번호**를 입력하고 **로그인** 클릭
3. "✅ ○○○님, 환영합니다!" 가 뜨면 성공! 🎉

**틀린 비밀번호 테스트:** 일부러 비밀번호를 틀리게 넣어보세요. "❌ 아이디 또는 비밀번호가 올바르지 않습니다." 가 뜨면 잘 막고 있는 거예요.

**토큰(손목밴드) 눈으로 보기:** 로그인 성공 후, 브라우저에서 `F12`를 누르고 상단에서 **Application**(또는 애플리케이션) 탭 → 왼쪽 **Local Storage** → `http://localhost:3000` 을 클릭하면, `token`이라는 항목에 `xxxxx.yyyyy.zzzzz` 처럼 점 두 개로 나뉜 긴 문자열이 보입니다. 이게 바로 방금 발급받은 토큰이에요.
