# 12단계 (1) — 자료 등록 API (Material + Copy 함께 저장)

> 목표: 관리자가 자료 하나(Material)와 그 자료의 실물 부수 여러 개(Copy)를
> **한 번의 요청으로 함께** 저장할 수 있는 창구를 주방(NestJS)에 만듭니다.
> 소요 시간: 약 40분

---

## 오늘의 새 개념: 트랜잭션 (한 묶음 처리)

자료 A를 등록하면서 부수 A-1, A-2, A-3도 함께 넣어야 하죠. 그런데 만약 자료는 잘 저장됐는데 A-2 부수 저장 도중 에러가 나면? "자료는 있는데 부수가 반쪽만" 있는 이상한 상태가 남게 돼요.

이걸 막는 장치가 **트랜잭션(transaction)** 입니다. "한 묶음의 저장을 모두 성공하든 모두 실패하든, 반쪽 상태는 절대 만들지 않는" 안전장치예요. 사서 실무로 치면 "자료 하나에 부수 세 개를 등록하는 그 작업 전체가 한 세트"이니, 이걸 그대로 코드로 지키는 거죠. Prisma는 `prisma.$transaction(...)` 이라는 도구로 이 일을 해줍니다.

이번 단계는 **주방 API만** 만들고, 실제 등록 화면은 다음(12-2)에서 만듭니다.

준비: 두 서버를 켜두세요.

## 2. 주방(NestJS)에 자료 등록 서비스 만들기 — 약 15분

`backend` → `src` 폴더에 **`materials`** 폴더를 새로 만들고, 그 안에 **`materials.service.ts`** 파일을 만들어 붙여넣습니다.

```typescript
import {
  Injectable,
  BadRequestException,
  ConflictException,
} from '@nestjs/common';
import { PrismaService } from '../prisma.service';

// 자료 종류 검증용 목록 (프론트와 동일하게 유지)
const ALLOWED_TYPES = [
  'book', 'dvd', 'article', 'thesis', 'law', 'video', 'music',
  'webpage', 'boardgame', 'tool', 'equipment', 'collection',
  'photo', 'clipping', 'etc',
];

// 자료 하나에 딸린 부수 정보의 모양
type CopyInput = {
  registrationNo: string;
  specialCode?: string;
  shelfNo?: string;
  location?: string;
};

// 등록 화면에서 넘어오는 정보의 모양
type CreateMaterialData = {
  type: string;
  title: string;
  creator?: string;
  publisher?: string;
  pubYear?: string;
  isbn?: string;
  callNumber?: string;
  summary?: string;
  coverUrl?: string;
  onlineUrl?: string;
  marcRaw?: string;
  extra?: any;
  copies?: CopyInput[]; // 실물 부수 여러 개
};

@Injectable()
export class MaterialsService {
  constructor(private prisma: PrismaService) {}

  async createWithCopies(libraryId: number, data: CreateMaterialData) {
    // 1) 기본 검증
    if (!data.type || !data.title) {
      throw new BadRequestException('종류와 제목은 필수입니다.');
    }
    if (!ALLOWED_TYPES.includes(data.type)) {
      throw new BadRequestException('알 수 없는 자료 종류입니다.');
    }

    // 2) 부수(등록번호) 중복 사전 검사 — 같은 도서관 안에서만
    const copies = data.copies ?? [];
    // 화면에서 온 부수들끼리도 중복 있는지
    const seen = new Set<string>();
    for (const c of copies) {
      if (!c.registrationNo) {
        throw new BadRequestException('부수의 등록번호는 필수입니다.');
      }
      if (seen.has(c.registrationNo)) {
        throw new ConflictException(
          `입력한 부수 중 등록번호가 중복됩니다: ${c.registrationNo}`,
        );
      }
      seen.add(c.registrationNo);
    }
    // 이미 DB에 있는 등록번호와 겹치는지
    if (copies.length > 0) {
      const existing = await this.prisma.copy.findMany({
        where: {
          libraryId,
          registrationNo: { in: copies.map((c) => c.registrationNo) },
        },
        select: { registrationNo: true },
      });
      if (existing.length > 0) {
        throw new ConflictException(
          `이미 사용 중인 등록번호: ${existing.map((e) => e.registrationNo).join(', ')}`,
        );
      }
    }

    // 3) 트랜잭션으로 "자료 + 부수들"을 한 묶음으로 저장
    //    (하나라도 실패하면 전체가 원상복구됩니다)
    return this.prisma.$transaction(async (tx) => {
      const material = await tx.material.create({
        data: {
          libraryId,
          type: data.type,
          title: data.title,
          creator: data.creator || undefined,
          publisher: data.publisher || undefined,
          pubYear: data.pubYear || undefined,
          isbn: data.isbn || undefined,
          callNumber: data.callNumber || undefined,
          summary: data.summary || undefined,
          coverUrl: data.coverUrl || undefined,
          onlineUrl: data.onlineUrl || undefined,
          marcRaw: data.marcRaw || undefined,
          extra: data.extra ?? undefined,
        },
      });

      if (copies.length > 0) {
        await tx.copy.createMany({
          data: copies.map((c) => ({
            libraryId,
            materialId: material.id,
            registrationNo: c.registrationNo,
            specialCode: c.specialCode || null,
            shelfNo: c.shelfNo || null,
            location: c.location || null,
          })),
        });
      }

      return { id: material.id, copiesCount: copies.length };
    });
  }
}
```

`this.prisma.$transaction(async (tx) => { ... })` 이 부분이 트랜잭션이에요. **중괄호 안에서 자료와 부수를 모두 저장**하는데, 도중에 에러가 나면 앞서 저장된 것들이 자동으로 원상복구됩니다. `tx`는 그 트랜잭션 안에서 쓰는 임시 통역사예요.

## 3. 등록 요청을 받는 창구 만들기 — 약 8분

같은 `materials` 폴더에 **`materials.controller.ts`** 파일을 만들어 붙여넣습니다.

```typescript
import { Body, Controller, Post, Req, UseGuards } from '@nestjs/common';
import { MaterialsService } from './materials.service';
import { AdminGuard } from '../auth/admin.guard';

@Controller('materials')
export class MaterialsController {
  constructor(private materialsService: MaterialsService) {}

  // POST /materials — 관리자만(문지기 통과 필요)
  @UseGuards(AdminGuard)
  @Post()
  create(@Req() req: any, @Body() body: any) {
    // AdminGuard가 확인 후 request.user에 넣어둔 정보에서 libraryId를 꺼냅니다.
    // (관리자가 자기 도서관 것만 만들 수 있게 자동으로 제한됨)
    const libraryId: number = req.user.libraryId;
    return this.materialsService.createWithCopies(libraryId, body);
  }
}
```

여기서 조용하지만 중요한 안전장치가 하나 있어요. **`libraryId`를 브라우저에서 받지 않고, 관리자 토큰에 담긴 값을 씁니다.** 이러면 A도서관 관리자가 실수(또는 장난)로 "B도서관에 자료를 등록해줘"라고 요청해도 무시돼요. 우리 멀티 도서관 원칙(모든 조회에 libraryId 필터 필수)이 이 지점에서도 지켜집니다.

## 4. 주방에 새 부품 등록 — 약 3분

`backend` → `src` → **`app.module.ts`** 를 열고, `Materials` 관련 두 줄과 임포트를 추가합니다. 전체를 아래로 통째로 바꿔주세요.

```typescript
import { Module } from '@nestjs/common';
import { JwtModule } from '@nestjs/jwt';
import { AppController } from './app.controller';
import { AppService } from './app.service';
import { PrismaService } from './prisma.service';
import { LibraryController } from './library/library.controller';
import { LibraryService } from './library/library.service';
import { UsersController } from './users/users.controller';
import { UsersService } from './users/users.service';
import { AuthController } from './auth/auth.controller';
import { AuthService } from './auth/auth.service';
import { MaterialsController } from './materials/materials.controller';
import { MaterialsService } from './materials/materials.service';

@Module({
  imports: [
    JwtModule.register({
      global: true,
      secret: process.env.JWT_SECRET,
      signOptions: { expiresIn: '7d' },
    }),
  ],
  controllers: [
    AppController,
    LibraryController,
    UsersController,
    AuthController,
    MaterialsController,
  ],
  providers: [
    AppService,
    PrismaService,
    LibraryService,
    UsersService,
    AuthService,
    MaterialsService,
  ],
})
export class AppModule {}
```

저장하면 주방 서버가 다시 시작됩니다. 에러 없이 `Nest application successfully started`가 나오는지 봐주세요.
