# 개선 (3) — 로그인 Enter 키 지원 + 원래 있던 페이지로 복귀

> 목표: 로그인 화면에서 아이디·비밀번호를 입력하고 **Enter**를 누르면 바로 로그인되게 하고,
> `/admin/materials/list` 같은 관리자 페이지에 로그인 없이 들어갔다가 로그인하면
> 홈이 아니라 **원래 가려던 그 페이지**로 돌아가게 고칩니다.
> 소요 시간: 약 15분

---

## 오늘의 핵심: 왜 홈으로 갔을까?

로그인 페이지는 이미 "로그인 성공하면 `redirect`라는 주소표를 보고 그리로 돌아간다"는 기능이 있어요. 문제는 **누가 그 주소표를 써주냐**예요.

- 홈 화면 사이드바의 "로그인" 버튼을 누르면 → `/login?redirect=/현재페이지` 처럼 **주소표를 챙겨서** 로그인 페이지로 갑니다. 이 경우는 잘 돌아와요.
- 그런데 지난번에 만든 `useRequireAuth`(로그인 안 하고 관리자 페이지에 들어가면 쫓아내는 문지기)는 그냥 `router.push("/login")`만 하고 있어서, **주소표를 안 챙겨줍니다.** 그래서 로그인 후 어디로 가야 할지 몰라 기본값인 홈으로 가는 거예요.

그래서 문지기(`useRequireAuth`)도 "지금 있던 곳"을 적어서 로그인 페이지로 보내주면 해결됩니다.

준비: 두 서버를 켜두세요.

---

## 1. 문지기가 주소표를 챙기게 고치기 — 약 5분

`frontend` → `src` → `hooks` → **`useRequireAuth.ts`** 를 아래로 통째로 바꿔주세요. (`usePathname`을 새로 가져와서, 쫓아낼 때 현재 주소를 `redirect`에 담아줍니다.)

```ts
"use client";

import { useEffect } from "react";
import { useRouter, usePathname } from "next/navigation";
import { useNotify } from "@/components/notify-provider";
import { useI18n } from "@/components/language-provider";

export function useRequireAuth() {
  const router = useRouter();
  const pathname = usePathname();
  const { notify } = useNotify();
  const { t } = useI18n();

  useEffect(() => {
    const token = localStorage.getItem("token");
    if (!token) {
      notify("❌ " + t("auth.loginRequired"), "error");
      router.push(`/login?redirect=${encodeURIComponent(pathname)}`);
    }
    // eslint-disable-next-line react-hooks/exhaustive-deps
  }, []);
}
```

바뀐 부분은 딱 두 곳이에요.

- `usePathname()` 을 새로 불러와서 지금 보고 있는 주소(`/admin/materials/list` 같은 것)를 알아냅니다.
- `router.push("/login")` → `router.push(\`/login?redirect=${encodeURIComponent(pathname)}\`)` 로 바뀌었어요. `encodeURIComponent`는 주소에 특수문자가 있어도 안전하게 포장해주는 역할입니다.

## 2. 로그인 화면에서 Enter 키로도 제출되게 고치기 — 약 8분

`(site)` → `login` → **`page.tsx`** 를 아래로 통째로 바꿔주세요. (입력칸 두 개를 `<form>` 태그로 감싸는 게 핵심이에요. 브라우저는 `<form>` 안에서 Enter를 누르면 자동으로 "제출"로 인식합니다.)

```tsx
"use client";

import { useState } from "react";
import { useRouter } from "next/navigation";
import { Button } from "@/components/ui/button";
import { Input } from "@/components/ui/input";
import { Label } from "@/components/ui/label";
import { Card, CardContent, CardHeader, CardTitle } from "@/components/ui/card";
import { useI18n } from "@/components/language-provider";
import { useNotify } from "@/components/notify-provider";
import { useAuth } from "@/components/auth-provider";

const API_URL = "http://localhost:3001";

export default function LoginPage() {
  const { t } = useI18n();
  const { notify } = useNotify();
  const { login } = useAuth();
  const router = useRouter();
  const [loginId, setLoginId] = useState("");
  const [password, setPassword] = useState("");

  async function handleLogin() {
    const res = await fetch(`${API_URL}/auth/login`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ loginId, password }),
    });
    if (res.ok) {
      const data = await res.json();
      login({ token: data.token, userName: data.user.name, role: data.user.role });
      notify(t("login.welcome").replace("{name}", data.user.name), "success");
      const redirect = new URLSearchParams(window.location.search).get("redirect") || "/";
      router.push(redirect);
    } else {
      const data = await res.json().catch(() => null);
      notify("❌ " + (data?.message || t("login.fail")), "error");
    }
  }

  function handleSubmit(e: React.FormEvent) {
    e.preventDefault();
    handleLogin();
  }

  return (
    <main className="mx-auto max-w-md p-8">
      <Card>
        <CardHeader>
          <CardTitle>{t("login.title")}</CardTitle>
        </CardHeader>
        <CardContent>
          <form onSubmit={handleSubmit} className="flex flex-col gap-4">
            <div className="flex flex-col gap-2">
              <Label htmlFor="loginId">{t("login.id")}</Label>
              <Input
                id="loginId"
                value={loginId}
                onChange={(e) => setLoginId(e.target.value)}
              />
            </div>
            <div className="flex flex-col gap-2">
              <Label htmlFor="password">{t("login.password")}</Label>
              <Input
                id="password"
                type="password"
                value={password}
                onChange={(e) => setPassword(e.target.value)}
              />
            </div>
            <Button type="submit" className="cursor-pointer">
              {t("login.submit")}
            </Button>
          </form>
        </CardContent>
      </Card>
    </main>
  );
}
```

바뀐 부분 정리:

- 아이디·비밀번호 칸과 버튼을 `<form onSubmit={handleSubmit}>` 로 감쌌어요.
- `handleSubmit`이라는 작은 함수를 새로 만들었어요. `e.preventDefault()`는 "브라우저야, 페이지를 새로고침하지 말고 내가 시키는 대로만 해"라는 뜻이에요(원래 `<form>`은 제출되면 페이지가 새로고침되려고 하는데, 그걸 막는 필수 코드입니다).
- 버튼은 `onClick={handleLogin}` 대신 `type="submit"`으로 바뀌었어요. `<form>` 안에서 `type="submit"` 버튼은 클릭해도, Enter를 눌러도 똑같이 `<form>`의 `onSubmit`을 실행시켜 줍니다. 그래서 버튼 클릭과 Enter 키가 이제 똑같이 동작해요.

> 만약 지금 로그인 화면 코드가 위 코드와 많이 다르게 되어 있다면(직접 수정하신 부분이 있다면), 지금 갖고 계신 코드를 저에게 보여주세요 — 그걸 기준으로 다시 맞춰 드릴게요.

## 3. 확인하기 — 약 5분

1. 로그아웃 상태에서 브라우저 주소창에 직접 `http://localhost:3000/admin/materials/list` 를 입력해서 들어갑니다.
2. "로그인이 필요합니다" 모달이 뜨고 로그인 페이지로 이동하는지 확인합니다. 이때 브라우저 주소창을 보면 `http://localhost:3000/login?redirect=%2Fadmin%2Fmaterials%2Flist` 처럼 끝에 `?redirect=...`가 붙어 있어야 정상이에요.
3. 아이디 칸에 아이디를 입력하고, **Tab** 키(또는 클릭)로 비밀번호 칸으로 이동해 비밀번호를 입력한 뒤, 버튼을 누르지 않고 **Enter** 키만 눌러보세요. 로그인이 바로 되면 성공입니다.
4. 로그인 성공 후 `http://localhost:3000/admin/materials/list` 로 돌아왔는지 확인하세요(홈으로 가면 안 됩니다).
5. 마지막으로, 홈 화면 사이드바의 "로그인" 버튼으로 들어가서 로그인했을 때도 원래대로 잘 동작하는지(이전 페이지로 복귀) 확인하세요 — 이 경로는 원래도 잘 되던 것이니 계속 잘 돼야 정상입니다.

✅ 다 확인되면 성공입니다! 🎉
