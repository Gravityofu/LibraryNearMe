# 2단계 — 프로젝트의 두 엔진 만들기

> 목표: 시스템의 두 축인 **NestJS(뒤에서 일하는 서버)** 와 **Next.js(사람이 보는 화면)** 를 만들어서,
> 브라우저에 실제로 첫 화면을 띄웁니다.
> 소요 시간: 약 40분

---

## 먼저, 오늘 만들 두 가지를 식당에 비유하면

우리 시스템을 식당이라고 생각해 보세요.

| 부분 | 이름 | 하는 일 | Apache 시절로 치면 |
|------|------|---------|-------------------|
| **홀 (손님 공간)** | **Next.js** | 손님이 보는 화면을 그림 (홈페이지·관리자 화면) | 브라우저에 보이던 HTML 부분 |
| **주방** | **NestJS** | 주문을 받아 데이터를 요리해서 내보냄 | PHP가 하던 서버 처리 |
| **창고** | **PostgreSQL** | 재료(책·회원·대출 기록)를 보관 | 데이터베이스 (그대로) |

⚠️ **이름이 비슷해서 헷갈립니다! 딱 한 번만 외워두세요.**
- **Ne*s*t (네스트)** = 주방 = 뒤에서 일하는 서버
- **Ne*x*t (넥스트)** = 홀 = 사람이 보는 화면

오늘은 **주방(NestJS)** 과 **홀(Next.js)** 을 각각 만들어 불을 켜봅니다. 이 둘을 서로 연결하는 것과 창고(DB) 연결은 3단계에서 합니다.

## 완성 후 폴더 모습

2단계가 끝나면 `C:\projects\LibraryNearMe` 폴더가 이렇게 됩니다.

```
LibraryNearMe/
├── backend/     ← NestJS (주방)
├── frontend/    ← Next.js (홀)
├── docs/        ← 우리가 만든 가이드들
└── README.md
```

## 1. NestJS 만들기 (주방) — 약 10분

### 1-1. NestJS 도구 설치

터미널을 열고 아래를 입력합니다. (이건 NestJS 프로젝트를 만들어주는 "공구"를 PC에 설치하는 것입니다. 한 번만 하면 됩니다.)

```
npm install -g @nestjs/cli
```

✅ 확인: `nest --version` 을 치면 버전 숫자가 나옵니다.

### 1-2. backend 프로젝트 만들기

```
cd C:\projects\LibraryNearMe
nest new backend --skip-git --package-manager npm
```

- `--skip-git`: backend 폴더 안에 별도의 Git 저장소가 또 생기는 걸 막습니다. (우리는 이미 맨 위 폴더에 Git이 있으니까요. 이걸 안 하면 "저장소 안에 저장소"가 겹쳐서 나중에 헷갈립니다.)
- 실행하면 파일들을 다운로드하느라 1~2분 걸립니다. 커피 한 모금 하세요 ☕

### 1-3. 포트 번호 바꾸기 (중요!)

NestJS와 Next.js는 둘 다 기본적으로 **3000번 문(포트)** 을 쓰려고 합니다. 같은 문을 둘이 동시에 쓸 수 없으니, 주방(NestJS)을 **3001번**으로 옮기겠습니다.

1. VS Code에서 `C:\projects\LibraryNearMe` 폴더를 엽니다. (파일 → 폴더 열기)
2. 왼쪽 파일 목록에서 `backend` → `src` → **`main.ts`** 파일을 클릭해 엽니다.
3. 아래와 비슷한 줄을 찾습니다. (숫자 `3000`이 들어 있는 줄)

   ```
   await app.listen(process.env.PORT ?? 3000);
   ```

4. `3000`을 **`3001`** 로 바꾸고 저장(`Ctrl+S`)합니다.

   ```
   await app.listen(process.env.PORT ?? 3001);
   ```

### 1-4. 주방에 불 켜기

```
cd C:\projects\LibraryNearMe\backend
npm run start:dev
```

여러 줄이 뜨다가 마지막에 `Nest application successfully started` 같은 문구가 나오면 성공입니다.

> 💡 **터미널이 멈춘 것처럼 보여도 정상입니다!** 서버가 "지금 손님을 기다리며 켜져 있는 상태"라서 그래요. 끄고 싶을 때는 그 터미널에서 `Ctrl+C`를 누르면 됩니다. **지금은 끄지 말고 켜 두세요.**

✅ **확인하기**: 브라우저 주소창에 아래를 입력하고 접속합니다.

```
http://localhost:3001
```

화면에 **`Hello World!`** 라고 보이면 주방이 켜진 것입니다! 🎉 (이 글자는 NestJS가 기본으로 넣어둔 인사말입니다.)

## 2. Next.js 만들기 (홀) — 약 15분

이제 **새 터미널을 하나 더 엽니다.** 방금 그 터미널은 주방이 계속 돌아가고 있으니 건드리지 말고, 두 번째 터미널을 새로 여세요.

> 터미널 앱에서 새 창(또는 새 탭)을 여는 방법: 터미널 앱 위쪽의 `+` 버튼을 누르거나, `Ctrl+Shift+동시에` 새 창을 엽니다. 헷갈리면 그냥 터미널 앱을 하나 더 실행해도 됩니다.

### 2-1. frontend 프로젝트 만들기

새 터미널에 아래를 입력합니다. **한 줄로 길게 입력**하세요 (복사해서 붙여넣기 추천).

```
cd C:\projects\LibraryNearMe
npx create-next-app@latest frontend --typescript --tailwind --eslint --app --src-dir --import-alias "@/*" --use-npm --disable-git
```

뒤에 붙은 것들은 "이런 설정으로 만들어 줘"라고 미리 정해준 것입니다. 뜻을 간단히 보면:
- `--typescript`: 실수를 줄여주는 안전장치가 있는 코드 방식
- `--tailwind`: 예쁘게 꾸미는 도구 (나중에 shadcn 디자인을 쓰려면 필요)
- `--app`, `--src-dir`: 요즘 표준 폴더 구조
- `--disable-git`: 여기도 별도 Git 저장소가 안 생기게 막기

> 만약 실행 중에 `Ok to proceed? (y)` 같은 질문이 나오면 `y`를 입력하고 Enter. 다른 질문이 나오면 그냥 Enter를 눌러 기본값을 받아들이면 됩니다. 설치에 2~3분 걸립니다.

### 2-2. 홀에 불 켜기

```
cd C:\projects\LibraryNearMe\frontend
npm run dev
```

`Ready` 또는 `Local: http://localhost:3000` 같은 문구가 나오면 켜진 것입니다.

✅ **확인하기**: 브라우저에서 아래에 접속합니다.

```
http://localhost:3000
```

Next.js 로고가 있는 **환영 화면**이 보이면 홀도 켜진 것입니다! 🎉

## 3. 지금 상태 정리

축하합니다. 지금 이런 상태가 되었습니다.

- 터미널 1번 창: **주방(NestJS)** 이 `localhost:3001`에서 돌아가는 중
- 터미널 2번 창: **홀(Next.js)** 이 `localhost:3000`에서 돌아가는 중
- 브라우저에서 두 주소 모두 각각 화면이 뜸

아직 주방과 홀은 서로 대화하지 않습니다. 각자 독립적으로 켜져만 있는 상태예요. 둘을 연결하는 건 3단계에서 합니다. 지금은 "두 엔진 모두 시동이 걸린다"를 확인한 것으로 충분합니다.

> 작업을 멈추고 싶을 때: 각 터미널에서 `Ctrl+C`를 누르면 서버가 꺼집니다. 다시 켜려면 해당 폴더에서 backend는 `npm run start:dev`, frontend는 `npm run dev`를 입력하면 됩니다.

## 4. GitHub에 저장하기 — 약 5분

오늘 만든 것을 인터넷 금고(GitHub)에 올려둡시다. 그래야 다른 PC에서도 이어서 작업할 수 있고, 실수로 지워도 복구됩니다.

**세 번째 터미널**을 새로 열거나, 서버가 도는 터미널 말고 다른 창에서 입력하세요. (서버 켜진 창에서 하면 안 됩니다.)

```
cd C:\projects\LibraryNearMe
git add .
git commit -m "2단계: NestJS와 Next.js 프로젝트 생성"
git push
```

> 혹시 `git push` 후에 뭔가 오래 걸리거나 에러가 나면, 그 메시지를 복사해서 저에게 알려주세요.

✅ **확인하기**: 브라우저에서 GitHub의 LibraryNearMe 페이지를 새로고침했을 때 `backend` 와 `frontend` 폴더가 새로 보이면 **2단계 완료입니다!** 🎉

> 💡 참고: `node_modules`라는 어마어마하게 큰 폴더는 GitHub에 안 올라갑니다. 이건 정상이에요. 이 폴더는 다른 PC에서 `npm install` 한 번이면 자동으로 다시 만들어지는, 말하자면 "언제든 다시 받을 수 있는 부품 창고"라서 금고에 넣지 않도록 이미 설정돼 있습니다.
