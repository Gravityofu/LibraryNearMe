# 개선 (5) — 최근 등록번호를 "도서관 전체 기준"으로 수정

> 목표: 등록번호 칸 옆의 "최근 등록" 표시가 **지금 보고 있는 자료의 최근 실물**이 아니라,
> **이 도서관에서 지금까지 부여한 모든 실물 중 가장 마지막 등록번호**를 보여주도록 고칩니다.
> 소요 시간: 약 15분

---

## 왜 지금은 다르게 나올까?

지난번(개선4)에 만든 코드는 `material.copies[0]`을 썼는데, 이건 **지금 화면에 열려 있는 그 자료(예: "노랜드")의 실물 목록** 중 첫 번째였어요. 즉 "이 책의 최근 등록번호"였지, "도서관 전체에서 가장 최근에 부여한 등록번호"가 아니었습니다.

말씀하신 대로 등록번호는 도서관 전체에서 겹치면 안 되는 값(중복 방지)이니, **어떤 자료를 보고 있든 상관없이 도서관 전체에서 가장 마지막 등록번호**를 알려줘야 다음 번호를 정하기 편해요. 이걸 하려면 백엔드에 "도서관에서 가장 최근 등록번호 하나만 알려줘" 하는 창구를 새로 하나 만들어야 합니다.

준비: 두 서버를 켜두세요.

---

# A. 주방(NestJS) 쪽 작업

## A-1. "가장 최근 등록번호 조회" 기능 추가 — 약 5분

`backend/src/materials/materials.service.ts` 를 엽니다. 맨 아래(`updateCopy` 함수 다음, 혹은 파일 맨 끝의 `}` 바로 위)에 아래 함수를 추가하세요.

```typescript
  async getLatestRegistrationNo(libraryId: number) {
    const latest = await this.prisma.copy.findFirst({
      where: { libraryId },
      orderBy: { createdAt: 'desc' },
      select: { registrationNo: true },
    });
    return { registrationNo: latest?.registrationNo ?? null };
  }
```

이 함수는 "이 도서관(libraryId)의 실물 중, 등록된 시각(`createdAt`)이 가장 최근인 것 하나만 찾아서 그 등록번호를 알려줘"라는 뜻이에요. 실물이 하나도 없으면 `null`을 돌려줍니다.

## A-2. 창구(라우트) 추가 — 약 3분

`backend/src/materials/copies.controller.ts` 를 엽니다. `@Get()` 으로 시작하는 `list` 함수 **바로 위**에 아래를 추가하세요.

```typescript

    return this.materialsService.getLatestRegistrationNo(req.user.libraryId);
  }

  @Get()

  list(@Req() req: any, @Query('page') page?: string, @Query('pageSize') pageSize?: string,
```

(아래 이어지는 `list` 함수 내용은 그대로 두시면 됩니다. `@Get('latest')` 블록만 새로 추가하는 거예요.)

전체 모양이 이렇게 되면 맞습니다.

```typescript
import { Controller, Get, Patch, Query, Param, Body, Req, UseGuards } from '@nestjs/common';
import { MaterialsService } from './materials.service';
import { AdminGuard } from '../auth/admin.guard';

@Controller('copies')
export class CopiesController {
  constructor(private materialsService: MaterialsService) {}

  @Get('latest')
  @UseGuards(AdminGuard)
  latest(@Req() req: any) {
    return this.materialsService.getLatestRegistrationNo(req.user.libraryId);
  }

  @Get()
  @UseGuards(AdminGuard)
  list(@Req() req: any, @Query('page') page?: string, @Query('pageSize') pageSize?: string,
    @Query('type') type?: string, @Query('title') title?: string, @Query('creator') creator?: string,
    @Query('subject') subject?: string, @Query('registrationNos') registrationNos?: string) {
    return this.materialsService.listCopies(req.user.libraryId, {
      page: page ? parseInt(page, 10) : 1,
      pageSize: pageSize ? parseInt(pageSize, 10) : 10,
      type, title, creator, subject,
      registrationNos: registrationNos ? registrationNos.split(',').map((s) => s.trim()).filter(Boolean) : undefined,
    });
  }

  @Patch(':id')
  @UseGuards(AdminGuard)
  update(@Req() req: any, @Param('id') id: string, @Body() body: any) {
    return this.materialsService.updateCopy(req.user.libraryId, parseInt(id, 10), body);
  }
}
```

저장하면 백엔드 서버가 재시작될 거예요. 문제없이 켜지는지 확인하세요.

`loadMaterial` 함수 **위나 아래 아무 곳**에 아래 함수를 추가하세요. (예: `refreshMaterial` 함수 바로 아래)

```tsx

    const token = localStorage.getItem("token");
    if (!token) return;
    const res = await fetch(`${API_URL}/copies/latest`, {
      headers: { Authorization: `Bearer ${token}` },
    });
    if (res.ok) {
      const data = await res.json();
      setLatestRegNo(data.registrationNo);
    }
  }
```
