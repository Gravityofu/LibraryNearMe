# 13단계 (4) — 사이드바 정리: "자료 등록"+"실물 등록" → "목록" 하나로

> 목표: 관리자 사이드바의 "자료 등록"·"실물 등록" 두 메뉴를 "목록" 메뉴 하나로 합칩니다.
> 자료 신규 등록·실물 등록 화면은 이제 "목록" 화면에서 버튼을 눌러야만 들어가는
> 하위 화면이 되니, 그 두 화면 오른쪽 위에 "뒤로 가기" 버튼도 넣습니다.
> 소요 시간: 약 15분

---

## 1. dictionary.ts에 새 메뉴 이름표 추가 — 약 2분

`frontend/src/lib/dictionary.ts` 를 엽니다. `"admin.menu.materialsCopy"` 줄 바로 아래(한글·영어 각각)에 추가하세요.

**한글 쪽:**

```ts
    "admin.menu.materialsList": "목록",
```

**영어 쪽:**

```ts
    "admin.menu.materialsList": "List",
```

> `admin.menu.materialsNew`·`admin.menu.materialsCopy`는 더 이상 사이드바에서 안 쓰지만, 지워도 안 지워도 상관없어요. 그냥 안 쓰이는 이름표로 남겨둬도 문제 없습니다.

---

## 2. 사이드바 메뉴 교체 — 약 3분

`frontend/src/app/admin/layout.tsx` 를 엽니다. 아래 두 개의 `<Link>` 를 찾으세요.

```tsx
          <Link href="/admin/materials/new" className={navClass("/admin/materials/new")}>
            {t("admin.menu.materialsNew")}
          </Link>
          <Link href="/admin/materials/copies" className={navClass("/admin/materials/copies")}>
            {t("admin.menu.materialsCopy")}
          </Link>
```

이 두 줄을 지우고, 아래 **한 줄짜리 링크**로 바꿔주세요.

```tsx
          <Link href="/admin/materials/list" className={navClass("/admin/materials/list")}>
            {t("admin.menu.materialsList")}
          </Link>
```

전체 `<nav>` 부분은 이런 모양이 됩니다.

```tsx
        <nav className="mt-6 flex flex-row flex-wrap gap-x-4 gap-y-2 text-sm md:flex-col md:gap-1">
          <Link href="/admin" className={navClass("/admin", true)}>
            {t("admin.menu.settings")}
          </Link>
          <Link href="/admin/materials/list" className={navClass("/admin/materials/list")}>
            {t("admin.menu.materialsList")}
          </Link>
          <span className="px-3 py-2.5 text-[rgba(249,246,240,0.6)]">{t("admin.menu.members")}</span>
          <span className="px-3 py-2.5 text-[rgba(249,246,240,0.6)]">{t("admin.menu.loans")}</span>
        </nav>
```

나머지 코드(`navClass`, `useRequireAuth()`, 하단 로그아웃 블록 등)는 그대로 두시면 됩니다.

---

## 3. "자료 등록" 화면에 뒤로 가기 버튼 추가 — 약 5분

이제 `/admin/materials/new`는 사이드바에서 바로 들어가는 화면이 아니라, "목록" 화면 맨 아래 **자료 등록** 버튼을 눌러야 들어가는 하위 화면이 됐어요. 우리가 정한 규칙대로 오른쪽 위에 뒤로 가기 버튼을 넣습니다. (`/admin/materials/copies`는 개선(4)에서 이미 넣어뒀어요.)

`frontend/src/app/admin/materials/new/page.tsx` 를 엽니다.

**① import 추가**

아래 줄을 찾으세요.

```tsx
import MarcEditor, { DEFAULT_FIELDS, MarcField } from "@/components/marc-editor";
```

바로 아래에 한 줄 추가하세요.

```tsx
import MarcEditor, { DEFAULT_FIELDS, MarcField } from "@/components/marc-editor";
import AdminBackButton from "@/components/admin-back-button";
```

**② 제목 옆에 버튼 배치**

아래 부분을 찾으세요.

```tsx
    <div className="mx-auto max-w-3xl">
      <h1 className="mb-4 text-lg font-bold">{t("materials.new.title")}</h1>
```

이렇게 바꿔주세요.

```tsx
    <div className="mx-auto max-w-3xl">
      <div className="mb-4 flex items-center justify-between">
        <h1 className="text-lg font-bold">{t("materials.new.title")}</h1>
        <AdminBackButton href="/admin/materials/list" />
      </div>
```

## 4. 확인하기 — 약 5분

1. 관리자 화면(`/admin`)에 들어가서, 왼쪽 사이드바에 이제 "도서관 설정"·**"목록"**·회원 관리(준비 중)·대출/반납(준비 중) 이렇게 나오는지 확인합니다. ("자료 등록"·"실물 등록"은 더 이상 안 보여야 정상이에요.)
2. **목록** 메뉴를 클릭하면 `/admin/materials/list` 화면으로 잘 이동하는지 확인합니다.
3. 목록 화면 맨 아래 **자료 등록** 버튼을 눌러 `/admin/materials/new`로 들어가서, 오른쪽 위에 **"← 뒤로 가기"** 버튼이 보이고 누르면 목록으로 돌아가는지 확인합니다.
4. 목록의 아무 줄에서 **등록** 버튼을 눌러 실물 등록 화면으로 들어가서, 여기도 뒤로 가기 버튼이 잘 있는지(개선4에서 이미 넣었어요) 다시 한번 확인합니다.
5. ENG 토글을 눌러 "목록" 메뉴 이름이 "List"로 바뀌는지 확인합니다.

✅ 모두 확인되면 13단계 전체(목록 화면 + 실물 등록 개편 + 사이드바 정리)가 완성입니다! 🎉
