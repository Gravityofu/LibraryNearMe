# 7단계 (1) — 사용자(User) 표 만들기

> 목표: 회원·관리자·슈퍼계정을 모두 담을 "사용자 표"를 데이터베이스에 만듭니다.
> 지금까지 논의한 설계(소속 도서관, 아이디, 회원증 번호, 이전 대비 등)를 전부 담아요.
> 소요 시간: 약 30분

---

## 오늘의 위치

7단계(계정 시스템)는 네 부분으로 나눠서 진행합니다. 오늘은 그 **첫 번째**예요.

1. **사용자 표 만들기 ← 오늘**
2. 회원가입 기능
3. 로그인 기능 (비밀번호 암호화 + 출입증 토큰)
4. 관리자 페이지에 자물쇠 걸기

오늘은 코드를 실행하는 기능은 없고, **데이터를 담을 그릇(표)을 만드는 것**까지입니다. 3단계에서 `Library` 표를 만들었던 것과 똑같은 작업이에요. 그때보다 칸이 많을 뿐입니다.

## 준비

backend 서버는 켜둬도 되고 꺼둬도 됩니다. 표를 만드는 작업은 터미널에서 명령어로 진행해요.

## 1. 설계도에 사용자 표 추가하기 — 약 15분

VS Code에서 `backend` → `prisma` → **`schema.prisma`** 를 엽니다.

이번엔 내용을 **전부 지우고** 아래 내용을 통째로 붙여넣은 뒤 저장(`Ctrl+S`)하세요. (맨 위 `generator`와 `datasource`는 기존과 같고, `Library`에 `domain` 칸이 하나 늘고, `User` 표와 목록(enum)이 새로 추가된 것입니다.)

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// 도서관 (여러 도서관이 한 시스템을 함께 씁니다)
model Library {
  id           Int      @id @default(autoincrement())
  name         String
  logoUrl      String?
  primaryColor String   @default("#2563eb")
  domain       String?  @unique   // 이 도서관 홈페이지 도메인 (나중에 도메인별 서비스용)
  createdAt    DateTime @default(now())

  users        User[]             // 이 도서관에 속한 사용자들 (연결고리)
}

// 사용자 (회원 / 관리자 / 슈퍼계정 모두 여기에 담습니다)
model User {
  id            Int        @id @default(autoincrement())

  libraryId     Int                          // 어느 도서관 소속인지
  library       Library    @relation(fields: [libraryId], references: [id])

  loginId       String?                      // 아이디(도서관 내 고유). 이전 회원은 나중에 설정 → 비어도 됨
  passwordHash  String?                      // 암호화된 비밀번호. 소셜·이전 회원 대비 비어도 됨

  name          String                       // 이름 (필수)
  phone         String?                      // 휴대폰 번호. 중복 허용 (가입 화면에서 입력받음)
  phoneVerified Boolean    @default(false)    // 번호 인증 여부 (나중 문자 인증용)
  email         String?                      // 이메일 (선택)
  birthDate     DateTime?                     // 생년월일 (선택)
  address       String?                       // 주소 (선택)
  postalCode    String?                       // 우편번호 (선택)

  memberNo      String?                       // 회원증 번호(사람이 읽는). 기존 "U012345" 그대로 or 신규 발급
  cardToken     String     @unique            // QR 보안 코드(기계용). 무작위 고유값

  role          Role       @default(MEMBER)   // 역할
  status        UserStatus @default(ACTIVE)   // 계정 상태

  provider      String?                       // 소셜 로그인 경로 (나중용, 지금 비움)
  providerId    String?                       // 소셜 계정 식별값 (나중용, 지금 비움)

  createdAt     DateTime   @default(now())

  @@unique([libraryId, loginId])              // 같은 도서관 안에서 아이디 중복 금지
  @@unique([libraryId, memberNo])             // 같은 도서관 안에서 회원증번호 중복 금지
}

// 역할: 아래 셋 중 하나만 가질 수 있음
enum Role {
  SUPER    // 슈퍼계정 (전체 관리)
  ADMIN    // 도서관 관리자
  MEMBER   // 일반 이용자
}

// 계정 상태
enum UserStatus {
  ACTIVE      // 활성
  PENDING     // 대기 (이전됐지만 아직 활성화 안 함 등)
  SUSPENDED   // 정지 / 탈퇴
}
```

새로 나온 개념 몇 가지만 짚을게요. (지금 다 외울 필요는 없어요.)

- **`?` 가 붙은 칸**(예: `loginId String?`)은 "비어 있어도 됨"이라는 뜻이에요. 이전 도서관 회원은 아직 우리 아이디·비밀번호가 없으니 비워둘 수 있어야 하거든요.
- **`enum`(목록)** 은 "정해진 값 중 하나만 넣을 수 있게" 제한하는 장치입니다. `role`에는 SUPER·ADMIN·MEMBER 셋만, `status`에는 ACTIVE·PENDING·SUSPENDED 셋만 들어가요. 오타로 엉뚱한 값이 저장되는 걸 막아줍니다.
- **`@relation` / `libraryId`** 는 "이 사용자가 어느 도서관 소속인지"를 도서관 표와 연결하는 고리예요. 우리 멀티 도서관 구조의 핵심이죠.
- **`@@unique([libraryId, loginId])`** 는 "같은 도서관 안에서는 아이디가 겹치면 안 되지만, 다른 도서관끼리는 같은 아이디도 괜찮다"는 규칙입니다. 앞서 논의한 그대로예요.

## 2. 설계도대로 진짜 표 만들기 (마이그레이션) — 약 5분

터미널에서 backend 폴더로 가서 실행합니다.

```
cd C:\projects\LibraryNearMe\backend
npx prisma migrate dev --name add_user
```

- `--name add_user`: 이 작업에 "사용자 표 추가"라는 이름표를 붙입니다.
- Prisma가 창고에 `User` 표를 실제로 만들고, `Library` 표에는 `domain` 칸을 더하고, 통역사(Prisma Client)도 새 표를 알도록 갱신합니다.

✅ **확인하기**: `Your database is now in sync with your schema` 같은 문구가 나오고, 에러가 없으면 성공입니다.

> 혹시 빨간 에러가 나면 그 메시지를 복사해서 보내주세요. 특히 오타(칸 이름 등)로 인한 문법 오류가 흔하니, 1번의 붙여넣기가 정확한지 먼저 확인해 보시면 좋아요.

## 3. 표가 잘 생겼는지 눈으로 확인하기 — 약 5분

```
npx prisma studio
```

브라우저에 Prisma Studio(`localhost:5555`)가 열립니다.

✅ **확인하기**: 왼쪽 목록에 **`User`** 가 새로 보이고, 클릭하면 우리가 설계한 칸들(loginId, name, phone, memberNo, cardToken, role ...)이 쭉 나타나면 성공입니다! (아직 회원이 없으니 내용은 비어 있는 게 정상이에요.) `Library`를 클릭하면 `domain` 칸도 새로 생긴 걸 볼 수 있어요.

확인했으면 Prisma Studio 터미널에서 `Ctrl+C`로 꺼주세요.

> 이번 단계에서는 일부러 회원을 넣지 않습니다. 회원을 제대로(비밀번호를 안전하게 암호화해서) 만드는 건 다음 "회원가입" 부분에서 할 거예요.
