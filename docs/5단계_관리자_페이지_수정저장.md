# 5단계 — 관리자 페이지에서 도서관 설정 수정·저장하기

> 목표: 웹 화면(관리자 페이지)에서 도서관 이름과 색상을 직접 입력하고 저장합니다.
> 이제 Prisma Studio 없이도 설정을 바꿀 수 있어요.
> 소요 시간: 약 45분

---

## 오늘의 그림

지금까지 데이터는 창고 → 주방 → 홀 → 화면, 이렇게 **한 방향**으로만 흘렀습니다(읽기). 오늘은 **반대 방향**을 뚫습니다. 관리자가 화면에서 입력한 값이 → 주방 → 창고로 저장되게(쓰기) 만드는 거예요.

```
[관리자 화면에서 입력]  →저장 요청→  [주방 NestJS]  →기록→  [창고 DB]
```

오늘 할 일은 크게 두 덩어리입니다.
- **A. 주방 쪽**: "설정을 바꿔 저장하는 창구"를 새로 만들고, 브라우저가 주방을 부를 수 있게 문을 엽니다(CORS).
- **B. 홀 쪽**: `/admin` 이라는 관리자 페이지를 만들어, 입력창과 저장 버튼을 놓습니다.

### 오늘 만나는 CORS

4단계에서 미뤄둔 CORS를 오늘 만납니다. 이번엔 **손님의 브라우저가 직접 주방(3001)에 전화**를 겁니다(저장 요청). 브라우저는 보안상, 지금 보고 있는 주소(3000)와 다른 주소(3001)로 함부로 전화하는 걸 막아요. 그래서 주방이 "3000에서 오는 전화는 받아도 돼"라고 허락해줘야 하는데, 그게 바로 CORS 설정입니다. 명령 한 줄이면 됩니다.

# A. 주방(NestJS) 쪽 작업

## A-1. 브라우저 전화를 받도록 문 열기 (CORS) — 약 5분

VS Code에서 `backend` → `src` → **`main.ts`** 를 엽니다. `bootstrap` 함수 안, `await app.listen(...)` **바로 윗줄**에 `app.enableCors();` 한 줄을 추가하세요. 아래 모양이 되면 됩니다.

```typescript
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  app.enableCors(); // 브라우저(3000)가 주방(3001)을 부를 수 있게 허용
  await app.listen(process.env.PORT ?? 3001);
}
bootstrap();
```

저장하면 주방 서버가 자동으로 다시 시작됩니다. (start:dev는 파일이 바뀌면 알아서 재시작해요.)

> 지금은 모든 곳에서 오는 전화를 다 받도록 열어뒀습니다. 개발 중엔 편해서 괜찮고, 나중에 배포할 때 "우리 홈페이지 주소만 허용"으로 좁히는 방법을 안내하겠습니다.

## A-2. 저장하는 일꾼 만들기 — 약 5분

`backend` → `src` → `library` → **`library.service.ts`** 를 엽니다. 내용을 아래로 통째로 바꿔주세요. (기존 `getLibrary` 아래에 `updateLibrary`가 추가된 것입니다.)

```typescript
import { Injectable } from '@nestjs/common';
import { PrismaService } from '../prisma.service';

@Injectable()
export class LibraryService {
  constructor(private prisma: PrismaService) {}

  // 도서관 설정 가져오기 (기존 기능)
  getLibrary() {
    return this.prisma.library.findFirst();
  }

  // 도서관 설정 바꿔서 저장하기 (오늘 추가)
  async updateLibrary(data: { name: string; primaryColor: string }) {
    // 먼저 우리 도서관을 찾고,
    const library = await this.prisma.library.findFirst();
    if (!library) return null;

    // 그 도서관의 이름과 색상을 새 값으로 바꿔 저장합니다.
    return this.prisma.library.update({
      where: { id: library.id },
      data: {
        name: data.name,
        primaryColor: data.primaryColor,
      },
    });
  }
}
```

`prisma.library.update(...)` 이 부분이, PHP 시절 `UPDATE library SET ... WHERE id = ...` 와 같은 일을 합니다.

## A-3. 저장 요청을 받는 창구 만들기 — 약 5분

같은 폴더의 **`library.controller.ts`** 를 열고, 아래로 통째로 바꿔주세요.

```typescript
import { Controller, Get, Patch, Body } from '@nestjs/common';
import { LibraryService } from './library.service';

@Controller('library')
export class LibraryController {
  constructor(private libraryService: LibraryService) {}

  // GET 요청: 설정 가져오기 (기존)
  @Get()
  getLibrary() {
    return this.libraryService.getLibrary();
  }

  // PATCH 요청: 설정 바꾸기 (오늘 추가)
  // 화면에서 보낸 이름·색상이 body 안에 담겨 옵니다.
  @Patch()
  updateLibrary(@Body() body: { name: string; primaryColor: string }) {
    return this.libraryService.updateLibrary(body);
  }
}
```

> `@Get`은 "가져오기", `@Patch`는 "일부 바꾸기"라는 요청 종류입니다. 같은 `/library` 주소라도 요청 종류에 따라 다른 창구가 응답해요. 맨 윗줄 `import`에 `Patch, Body`가 추가된 것도 확인하세요.

주방 쪽 작업 끝! 저장하면 서버가 다시 시작됩니다. 에러 없이 `successfully started`가 나오는지 봐주세요.

# B. 홀(Next.js) 쪽 작업 — 관리자 페이지 만들기

## B-1. /admin 페이지 만들기 — 약 15분

관리자 페이지는 새 주소 `/admin` 에 만듭니다. Next.js는 **폴더 이름이 곧 주소**가 되는 규칙이 있어요. 그래서 `admin` 폴더를 만들고 그 안에 `page.tsx`를 두면, 자동으로 `localhost:3000/admin` 주소가 생깁니다.

1. VS Code에서 `frontend` → `src` → `app` 폴더 위에서 마우스 오른쪽 클릭 → **새 폴더(New Folder)** → 이름을 **`admin`** 으로 만듭니다.
2. 방금 만든 `admin` 폴더 안에 **새 파일(New File)** → 이름을 **`page.tsx`** 로 만듭니다.
3. 그 파일에 아래 내용을 붙여넣고 저장합니다.

```tsx
// 맨 윗줄의 "use client"는 "이 화면은 손님의 브라우저에서 움직인다"는 표시입니다.
// 입력창에 타이핑하고 버튼을 누르는 것처럼, 사용자와 상호작용하는 화면에 붙입니다.
"use client";

import { useEffect, useState } from "react";

const API_URL = "http://localhost:3001";

export default function AdminPage() {
  // 입력창에 담길 값들을 기억하는 상자들입니다.
  const [name, setName] = useState("");
  const [primaryColor, setPrimaryColor] = useState("#2563eb");
  const [status, setStatus] = useState("");

  // 페이지가 처음 열릴 때, 현재 저장된 설정을 불러와 입력창을 채웁니다.
  useEffect(() => {
    fetch(`${API_URL}/library`)
      .then((res) => res.json())
      .then((data) => {
        if (data) {
          setName(data.name);
          setPrimaryColor(data.primaryColor);
        }
      });
  }, []);

  // 저장 버튼을 눌렀을 때 실행됩니다.
  async function handleSave() {
    setStatus("저장 중...");
    const res = await fetch(`${API_URL}/library`, {
      method: "PATCH", // "일부 바꾸기" 요청
      headers: { "Content-Type": "application/json" }, // "JSON 형식으로 보낼게요"
      body: JSON.stringify({ name, primaryColor }), // 실제로 보내는 값
    });
    if (res.ok) {
      setStatus("✅ 저장되었습니다! 홈페이지를 새로고침해서 확인해 보세요.");
    } else {
      setStatus("❌ 저장에 실패했습니다.");
    }
  }

  return (
    <main
      style={{
        maxWidth: "480px",
        margin: "3rem auto",
        padding: "2rem",
        fontFamily: "sans-serif",
        display: "flex",
        flexDirection: "column",
        gap: "1rem",
      }}
    >
      <h1 style={{ fontSize: "1.8rem", fontWeight: "bold" }}>
        도서관 설정 (관리자)
      </h1>

      <label>
        도서관 이름
        <input
          value={name}
          onChange={(e) => setName(e.target.value)}
          style={{
            display: "block",
            width: "100%",
            padding: "0.5rem",
            marginTop: "0.3rem",
            border: "1px solid #ccc",
            borderRadius: "6px",
          }}
        />
      </label>

      <label>
        대표 색상
        <input
          type="color"
          value={primaryColor}
          onChange={(e) => setPrimaryColor(e.target.value)}
          style={{
            display: "block",
            marginTop: "0.3rem",
            width: "60px",
            height: "40px",
            border: "1px solid #ccc",
            borderRadius: "6px",
          }}
        />
      </label>

      <button
        onClick={handleSave}
        style={{
          padding: "0.7rem",
          background: "#2563eb",
          color: "white",
          border: "none",
          borderRadius: "6px",
          cursor: "pointer",
          fontSize: "1rem",
        }}
      >
        저장하기
      </button>

      <p style={{ color: "#555" }}>{status}</p>
    </main>
  );
}
```

`type="color"` 로 만든 입력창은 누르면 **색상 선택기(팔레트)**가 뜹니다. 마우스로 색을 고를 수 있어요.

## C. 전체 흐름 확인하기 (오늘의 하이라이트!) — 약 5분

이제 처음부터 끝까지 연결해서 테스트합니다.

1. 브라우저에서 관리자 페이지 접속: `http://localhost:3000/admin`
2. 화면에 **현재 도서관 이름과 색상**이 입력창에 채워져 있는지 봅니다. (채워져 있으면 "불러오기"가 잘 된 것)
3. 도서관 이름을 바꿔보고, 색상 선택기로 색도 바꿉니다.
4. **저장하기** 버튼을 누릅니다. → "✅ 저장되었습니다!" 가 뜨면 성공.
5. 새 탭에서 홈페이지 접속: `http://localhost:3000` → 새로고침(F5)

✅ **확인하기**: 홈페이지의 도서관 이름과 색상이 방금 관리자 페이지에서 바꾼 값으로 변해 있으면 **대성공입니다!** 🎉🎉

방금 조제님은 "화면 입력 → 저장 → 데이터베이스 기록 → 다시 화면 표시"라는, 웹 시스템의 가장 기본이 되는 완전한 한 바퀴를 만든 거예요. 이제 이 시스템은 진짜 "관리 가능한" 시스템이 되었습니다.

> 저장 버튼을 눌렀는데 실패하거나 아무 반응이 없으면: (1) 주방 서버(터미널 1)가 켜져 있는지, (2) A-1의 `app.enableCors();`를 넣었는지 확인하세요. 브라우저 화면에서 `F12`를 눌러 나오는 개발자 도구의 `Console` 탭에 빨간 글씨가 있으면 그 내용을 복사해서 알려주세요.
