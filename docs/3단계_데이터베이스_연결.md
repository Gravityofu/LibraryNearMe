# 3단계 — 주방(NestJS)을 창고(PostgreSQL)에 연결하기

> 목표: 데이터베이스에 "도서관 설정" 표를 만들고, NestJS로 그 데이터를 꺼내
> 브라우저에서 확인합니다. 드디어 진짜 데이터를 다룹니다!
> 소요 시간: 약 50분

---

## 오늘의 그림

우리는 **Prisma**라는 도구를 새로 씁니다. Prisma는 주방(NestJS)과 창고(PostgreSQL) 사이의 **통역사**예요.

지금까지 조제님은 PHP에서 SQL 문장(`SELECT * FROM ...`)을 직접 써서 데이터베이스와 대화하셨죠? Prisma를 쓰면 SQL을 몰라도 `prisma.library.findFirst()` 같은 쉬운 자바스크립트 문장으로 데이터를 다룰 수 있습니다. 통역사가 알아서 SQL로 바꿔주거든요.

오늘 할 일을 순서대로 보면:
1. 창고에 `library`라는 이름의 방(데이터베이스)을 만든다
2. 주방에 통역사(Prisma)를 들인다
3. "도서관 설정" 표의 모양을 설계한다
4. 그 설계대로 창고에 실제 표를 만든다
5. 표에 데이터 한 줄을 눈으로 보며 넣는다
6. 주방에 "도서관 설정 좀 꺼내줘"라는 창구를 만든다
7. 브라우저에서 확인한다

## 준비 — 두 서버는 잠깐 꺼도 됩니다

2단계에서 켜둔 서버가 돌고 있다면, 이번 작업 동안은 backend 서버를 자주 껐다 켜야 합니다. 지금은 편하게 진행하기 위해, 서버가 도는 터미널에서 `Ctrl+C`를 눌러 잠깐 꺼두셔도 됩니다. (frontend는 켜두든 꺼두든 상관없어요.)

## 1. 창고에 방 만들기 (데이터베이스 생성) — 약 5분

터미널을 열고 PostgreSQL에 들어갑니다.

```
psql -U postgres
```

비밀번호를 입력하면 `postgres=#` 표시가 나옵니다. 여기서 아래를 입력하세요. (끝에 **세미콜론 `;`** 꼭 붙이세요. SQL의 마침표 같은 역할입니다.)

```
CREATE DATABASE library;
```

`CREATE DATABASE` 라고 응답이 나오면 성공입니다.

✅ **확인하기**: 아래를 입력하면 방 목록이 나오는데, `library`가 보이면 됩니다.

```
\l
```

목록에서 빠져나오려면 `q`를 누르고, psql에서 나가려면 `\q`를 입력합니다.

## 2. 주방에 통역사(Prisma) 들이기 — 약 10분

backend 폴더로 이동해서 Prisma를 설치합니다.

```
cd C:\projects\LibraryNearMe\backend
npm install prisma --save-dev
npm install @prisma/client
```

설치가 끝나면 Prisma의 기본 파일들을 만듭니다.

```
npx prisma init
```

이 명령이 backend 폴더 안에 두 가지를 만듭니다.
- `prisma` 폴더 → 그 안에 `schema.prisma` (표 설계도를 적는 파일)
- `.env` 파일 → 비밀번호 같은 민감한 정보를 적는 파일

### 2-1. 창고 주소를 .env에 적기

VS Code 왼쪽 파일 목록에서 `backend` → **`.env`** 파일을 엽니다. `DATABASE_URL=` 로 시작하는 줄이 있는데, 통째로 아래처럼 바꿔주세요.

```
DATABASE_URL="postgresql://postgres:여기에_비밀번호@localhost:5432/library?schema=public"
```

- `여기에_비밀번호` 자리에 PostgreSQL 설치할 때 정한 비밀번호를 넣으세요.
- 이 주소의 뜻: "postgres라는 사람이 / 이 비밀번호로 / 내 컴퓨터(localhost)의 5432번 문으로 들어가 / library 방을 쓴다"
- 저장(`Ctrl+S`) 하세요.

> ⚠️ 비밀번호에 `@`, `#`, `:`, `/` 같은 특수문자가 들어 있으면 주소가 깨질 수 있습니다. 그런 경우엔 저에게 알려주세요. 처리 방법을 따로 드릴게요.

> 🔒 **중요**: `.env` 파일에는 비밀번호가 들어 있어서 GitHub에 올라가면 안 됩니다. 1단계에서 넣어둔 설정 덕분에 자동으로 제외되지만, 나중에 4번 단계에서 직접 확인해 볼 거예요.

## 3. 표 설계도 그리기 — 약 5분

`backend` → `prisma` → **`schema.prisma`** 파일을 엽니다. 아래쪽에 이미 몇 줄(datasource, generator)이 적혀 있을 텐데, **그 아래에** 다음 내용을 추가하고 저장하세요.

```prisma
model Library {
  id           Int      @id @default(autoincrement())
  name         String
  logoUrl      String?
  primaryColor String   @default("#2563eb")
  createdAt    DateTime @default(now())
}
```

한 줄씩 뜻을 보면:
- `model Library` → "Library(도서관)라는 표를 만들 거야"
- `id ... @id @default(autoincrement())` → 각 줄의 고유 번호. 1, 2, 3... 자동으로 매겨짐
- `name String` → 도서관 이름 (글자)
- `logoUrl String?` → 로고 이미지 주소. 뒤의 **`?`** 는 "없어도 됨"이라는 뜻
- `primaryColor String @default("#2563eb")` → 대표 색상. 안 정하면 기본 파란색
- `createdAt DateTime @default(now())` → 이 줄이 만들어진 시각. 자동 기록

지침에 있던 "도서관마다 이름·로고·색상을 다르게" 하는 그 기능의 **첫 씨앗**입니다.

## 4. 설계도대로 창고에 진짜 표 만들기 (마이그레이션) — 약 5분

"마이그레이션(migration)"은 설계도를 실제 데이터베이스 표로 바꾸는 작업입니다. 터미널(backend 폴더)에서:

```
npx prisma migrate dev --name init
```

- `--name init`: 이 작업에 "init(처음 만들기)"이라는 이름표를 붙입니다.
- 실행되면 Prisma가 창고에 `Library` 표를 실제로 만들고, 통역사도 이 표를 알도록 업데이트합니다.

✅ **확인하기**: 터미널에 `Your database is now in sync with your schema` 같은 문구가 나오고, `backend/prisma/migrations` 폴더가 새로 생기면 성공입니다.

## 5. 표에 데이터 한 줄 넣기 (눈으로 보면서) — 약 5분

Prisma에는 데이터를 눈으로 보고 편집할 수 있는 **Prisma Studio**라는 예쁜 도구가 있습니다. pgAdmin보다 훨씬 간단해요.

```
npx prisma studio
```

브라우저가 자동으로 열리면서 `http://localhost:5555` 화면이 나옵니다. (안 열리면 직접 주소창에 입력)

1. 왼쪽 목록에서 **`Library`** 를 클릭합니다. (지금은 표가 비어 있어요.)
2. 오른쪽 위 **`Add record`** 버튼을 누릅니다.
3. `name` 칸에 도서관 이름을 입력하세요. 예: `무지개 도서관`
4. `primaryColor`는 그대로 두거나 원하는 색을 넣어도 됩니다. `logoUrl`은 비워둬도 됩니다.
5. 오른쪽 위 **`Save 1 change`** (초록 버튼)를 눌러 저장합니다.

✅ **확인하기**: 표에 도서관 이름이 한 줄 생기면 성공! 이제 창고에 실제 데이터가 들어간 것입니다.

확인했으면 이 터미널에서 `Ctrl+C`를 눌러 Prisma Studio를 꺼주세요.

## 6. 주방에 "도서관 설정 꺼내주는 창구" 만들기 — 약 10분

이제 NestJS 코드를 조금 씁니다. 세 개의 파일을 만들고 하나를 수정합니다. VS Code에서 진행하세요.

### 6-1. 통역사를 NestJS에 등록하는 파일

`backend/src` 폴더 안에 **`prisma.service.ts`** 라는 새 파일을 만들고, 아래 내용을 붙여넣습니다.

```typescript
import { Injectable, OnModuleInit } from '@nestjs/common';
import { PrismaClient } from '@prisma/client';

// 이 파일은 주방이 켜질 때 통역사(Prisma)와 연결을 맺어둡니다.
@Injectable()
export class PrismaService extends PrismaClient implements OnModuleInit {
  async onModuleInit() {
    await this.$connect();
  }
}
```

### 6-2. 도서관 데이터를 꺼내오는 파일

`backend/src` 안에 **`library`** 폴더를 새로 만들고, 그 안에 **`library.service.ts`** 파일을 만들어 아래를 붙여넣습니다.

```typescript
import { Injectable } from '@nestjs/common';
import { PrismaService } from '../prisma.service';

@Injectable()
export class LibraryService {
  constructor(private prisma: PrismaService) {}

  // 첫 번째 도서관 설정을 통째로 가져옵니다.
  getLibrary() {
    return this.prisma.library.findFirst();
  }
}
```

`prisma.library.findFirst()` 이 한 줄이, PHP 시절 `SELECT * FROM library LIMIT 1` 과 같은 일을 합니다.

### 6-3. 브라우저가 부를 수 있는 주소(창구)를 만드는 파일

같은 `library` 폴더 안에 **`library.controller.ts`** 파일을 만들어 아래를 붙여넣습니다.

```typescript
import { Controller, Get } from '@nestjs/common';
import { LibraryService } from './library.service';

// 이 창구의 주소는 /library 입니다.
@Controller('library')
export class LibraryController {
  constructor(private libraryService: LibraryService) {}

  // 누군가 http://localhost:3001/library 로 오면 이 함수가 실행됩니다.
  @Get()
  getLibrary() {
    return this.libraryService.getLibrary();
  }
}
```

### 6-4. 주방에 이 새 부품들을 등록하기

`backend/src` → **`app.module.ts`** 파일을 엽니다. 내용을 통째로 아래로 바꿔주세요. (기존 것과 거의 같고, 우리가 만든 3개가 추가된 것입니다.)

```typescript
import { Module } from '@nestjs/common';
import { AppController } from './app.controller';
import { AppService } from './app.service';
import { PrismaService } from './prisma.service';
import { LibraryController } from './library/library.controller';
import { LibraryService } from './library/library.service';

@Module({
  imports: [],
  controllers: [AppController, LibraryController],
  providers: [AppService, PrismaService, LibraryService],
})
export class AppModule {}
```

> `controllers`에는 "창구"들을, `providers`에는 "일꾼"들을 등록한다고 생각하면 됩니다. 우리가 만든 창구(LibraryController)와 일꾼(PrismaService, LibraryService)을 목록에 추가한 거예요.

## 7. 확인하기 — 브라우저에서 내 도서관 데이터 보기 — 약 5분

주방을 다시 켭니다.

```
cd C:\projects\LibraryNearMe\backend
npm run start:dev
```

에러 없이 `successfully started`가 나오면, 브라우저에서 아래에 접속하세요.

```
http://localhost:3001/library
```

✅ **확인하기**: 화면에 이런 모양의 글자가 나오면 대성공입니다! 🎉🎉

```
{"id":1,"name":"무지개 도서관","logoUrl":null,"primaryColor":"#2563eb","createdAt":"2026-..."}
```

이게 바로 **JSON**이라는 데이터 형식입니다. 방금 조제님은 "브라우저 요청 → NestJS 창구 → Prisma 통역 → PostgreSQL 창고 → 다시 브라우저"까지, 데이터가 한 바퀴 도는 전체 길을 완성한 거예요. 4단계에서는 이 데이터를 예쁜 홈페이지 화면에 띄웁니다.

> 에러가 나면? 주방 터미널에 빨간 글자가 뜰 텐데, 그 내용을 복사해서 저에게 보내주세요. 특히 `DATABASE_URL`이나 `PrismaClient` 관련 메시지면 .env 주소를 같이 확인해 봅시다.
