# 11단계 — 자료(Material) 표 + 부수(Copy) 표 만들기

> 목표: 조제님이 설명해주신 도서관 표준 구조(table1 자료 정보 / table2 실물 부수)를
> 그대로 데이터베이스 표로 만듭니다.
> 소요 시간: 약 35분

---

## 오늘의 그림

두 개의 표를 만들고 서로 연결합니다.

- **자료 표(Material)** = 조제님의 table1. 서지 정보(제목·저자·출판사·ISBN·청구기호·원본 KORMARC 등) 한 벌. 자료 A 하나.
- **부수 표(Copy)** = 조제님의 table2. 바코드 붙은 실물 한 권 한 권(A-1, A-2, A-3). 등록번호·별치기호·서가번호·소장처가 들어갑니다.

관계는 "자료 하나(A)에 부수 여러 개(A-1, A-2, A-3)"예요.

```
Material (자료 A)  ──┬── Copy A-1 (등록번호 1)
                     ├── Copy A-2 (등록번호 2)
                     └── Copy A-3 (등록번호 3)
```

- 온라인 자료(기사·웹페이지 등)는 부수가 0개일 수 있습니다(실물이 없으니까요).
- 종류마다 다른 특수 정보는 `extra`라는 상자(JSON)에, 원본 KORMARC는 `marcRaw` 칸에 담습니다.

이번 단계는 **표(그릇)만** 만드는 것까지예요. 자료를 실제로 등록하는 화면은 다음 단계에서 만듭니다.

준비: backend 서버는 켜둬도 꺼둬도 됩니다.

---

## 1. 설계도에 두 표 추가하기 — 약 15분

VS Code에서 `backend` → `prisma` → **`schema.prisma`** 를 엽니다.

### 1-1. Library 표에 연결고리 두 줄 추가

기존 `model Library { ... }` 를 찾아, `users User[]` 아래에 **두 줄**을 추가하세요.

```prisma
model Library {
  id           Int      @id @default(autoincrement())
  name         String
  logoUrl      String?
  primaryColor String   @default("#2563eb")
  domain       String?  @unique
  createdAt    DateTime @default(now())

  users        User[]
  materials    Material[]   // ← 추가
  copies       Copy[]       // ← 추가
}
```

### 1-2. 파일 맨 아래에 자료·부수 표와 상태 목록 추가

`schema.prisma` **맨 끝**에 아래 내용을 통째로 붙여넣습니다.

```prisma
// 자료 정보 (서지 레코드) — 조제님의 table1
model Material {
  id         Int      @id @default(autoincrement())

  libraryId  Int
  library    Library  @relation(fields: [libraryId], references: [id])

  type       String   // 자료 종류 코드 (book, dvd, article, ...) — 다음 단계에서 목록 연결
  title      String   // 서명(제목)
  creator    String?  // 저자 / 제작자
  publisher  String?  // 발행처(출판사)
  pubYear    String?  // 발행년 (문자열로: "2024" 등)
  isbn       String?  // ISBN
  callNumber String?  // 청구기호 (자료 단위)
  summary    String?  // 요약 / 설명
  coverUrl   String?  // 표지 이미지 주소
  onlineUrl  String?  // 온라인 자료 접근 주소
  marcRaw    String?  // 원본 KORMARC (도서·DVD만 채워짐)
  extra      Json?    // 종류별 특수 정보 (예: 보드게임 인원수)

  createdAt  DateTime @default(now())
  updatedAt  DateTime @updatedAt

  copies     Copy[]   // 이 자료에 속한 실물 부수들

  @@index([libraryId])
}

// 실물 개별 자료 (소장 부수) — 조제님의 table2
model Copy {
  id             Int        @id @default(autoincrement())

  libraryId      Int
  library        Library    @relation(fields: [libraryId], references: [id])
  materialId     Int
  material       Material   @relation(fields: [materialId], references: [id])

  registrationNo String     // 등록번호(고유번호) — 도서관 안에서 고유
  specialCode    String?    // 별치기호 (예: 아동, 그림)
  shelfNo        String?    // 서가번호
  location       String?    // 소장처 (예: 2층 문학)
  status         CopyStatus @default(AVAILABLE) // 상태

  createdAt      DateTime   @default(now())
  updatedAt      DateTime   @updatedAt

  @@unique([libraryId, registrationNo]) // 같은 도서관 안에서 등록번호 중복 금지
  @@index([materialId])
}

// 부수 상태 (대출/제적 기능 만들 때 더 다듬을 예정)
enum CopyStatus {
  AVAILABLE   // 비치(대출 가능)
  ON_LOAN     // 대출중
  RESERVED    // 예약 대기
  REPAIR      // 보수중
  LOST        // 분실
  WITHDRAWN   // 제적
}
```

새로 나온 것 두 가지만 짚을게요.
- `updatedAt DateTime @updatedAt` → 이 줄이 있으면, 이 자료를 수정할 때마다 **"마지막으로 고친 시각"이 자동으로 기록**됩니다. 편리해요.
- `@@index([libraryId])` → 나중에 "이 도서관의 자료만" 빠르게 찾도록 해주는 색인(찾아보기표)입니다. 검색이 많아질 자료 표라 미리 넣어둡니다.

저장하세요.

## 2. 설계도대로 표 만들기 (마이그레이션) — 약 5분

터미널에서 backend 폴더로 가서:

```
cd C:\projects\LibraryNearMe\backend
npx prisma migrate dev --name add_material_copy
```

✅ **확인하기**: `Your database is now in sync with your schema` 가 나오고 에러가 없으면 성공. (빨간 에러가 나면 대개 붙여넣기 중 오타이니, 메시지를 복사해 보내주세요.)

## 3. 표가 잘 생겼는지 + 연결 확인 — 약 10분

```
npx prisma studio
```

왼쪽 목록에 **`Material`** 과 **`Copy`** 표가 새로 보이면 성공입니다. 이왕이면 연결이 어떻게 되는지 직접 눈으로 확인해 볼까요? (선택이지만 이해에 큰 도움이 돼요.)

1. `Material` 표 → **Add record** → `libraryId`에 `1`(우리 도서관), `type`에 `book`, `title`에 아무 책 제목(예: "어린 왕자")을 넣고 **Save**. 방금 만든 자료의 `id`(예: 1)를 기억해 두세요.
2. `Copy` 표 → **Add record** → `libraryId`에 `1`, `materialId`에 방금 그 자료 id(예: 1), `registrationNo`에 `1`, `location`에 "2층 문학"을 넣고 **Save**.
3. 같은 방식으로 `registrationNo`가 `2`, `3`인 Copy를 두 개 더 만들어 보세요. (materialId는 똑같이!)

✅ **확인하기**: `Material`의 그 자료 줄을 열어보면 `copies` 자리에 방금 만든 부수 3개가 연결돼 보입니다. "자료 하나에 부수 여러 개"가 실제로 작동하는 거예요! 🎉

> 이건 연습용 데이터예요. 확인했으면 지워도 되고 그냥 둬도 됩니다. Prisma Studio는 `Ctrl+C`로 끕니다.
