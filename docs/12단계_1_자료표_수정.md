## 이번에 반영한 것 (요약)

- **분류기호 vs 청구기호 분리**: Material에는 `classNumber`(분류기호, MARC 056), Copy에는 `callNumber`(청구기호, 이용자가 자료 찾는 번호).
- **형태·주제어·언어**: Material에 `format`·`subject`·`language` 추가.
- **MARC 원본**: Material에 `marc`(편집본) + 기존 `marcRaw`(KOLIS 원본).
- **작업한 사서 기록**: Material·Copy에 `createdById`(등록한 사서) 추가, 사서 계정(User)과 연결.
- (Copy는 종류와 상관없이 "실물이 있는 모든 자료"에 등록합니다. 표 구조 변경은 없고 개념만 확정.)

준비: backend 서버는 켜둬도 꺼둬도 됩니다.

## 1. User 표에 연결고리 두 줄 추가 — 약 3분

`backend` → `prisma` → **`schema.prisma`** 를 엽니다. 기존 `model User { ... }` 안에서 관계 줄들(예: `library Library @relation(...)`) 근처, 닫는 `}` 위에 **두 줄**을 추가하세요.

```prisma
  createdMaterials Material[]   // 이 사서가 등록한 자료들
  createdCopies    Copy[]       // 이 사서가 등록한 실물들
```

(다른 User 칸들은 그대로 둡니다. 이 두 줄은 "이 사서가 등록한 것들"을 반대편에서 이어주는 고리예요.)

## 2. Material 표 교체 — 약 8분

기존 `model Material { ... }` 블록을 찾아 **아래로 통째로 바꿔주세요.**

```prisma
// 자료 정보 (서지 레코드)
model Material {
  id          Int      @id @default(autoincrement())

  libraryId   Int
  library     Library  @relation(fields: [libraryId], references: [id])

  type        String   // 자료 종류 코드 (book, dvd, thesis, ...)
  title       String   // 서명 — MARC 245 ▼a
  creator     String?  // 저자 — MARC 100(있으면) / 없으면 245 ▼d
  publisher   String?  // 발행처 — MARC 260 ▼b
  pubYear     String?  // 발행년 — MARC 260 ▼c
  isbn        String?  // ISBN — MARC 020 ▼a
  classNumber String?  // 분류기호 — MARC 056 (책·DVD 자동추출, 비도서는 선택 입력)
  format      String?  // 형태(크기·파일형식) — MARC 300
  subject     String?  // 주제어 — MARC 653
  language    String?  // 언어 — MARC 041
  summary     String?  // 요약/설명 — MARC 500 ▼a
  coverUrl    String?  // 표지 이미지 주소
  onlineUrl   String?  // 온라인 접근 주소

  marc        Json?    // 구조화된 MARC(태그 목록) — 책·DVD 서지 원본(편집본)
  marcRaw     String?  // KOLIS-NET 원본 문자열 그대로 보관
  extra       Json?    // 종류별 특수 정보

  createdById Int?     // 이 자료를 등록한 사서
  createdBy   User?    @relation(fields: [createdById], references: [id])

  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  copies      Copy[]

  @@index([libraryId])
}
```

> 이전 버전과 달라진 점: `callNumber` → `classNumber`(056)로 이름·의미 변경, `createdById`/`createdBy` 추가.

## 3. Copy 표 교체 — 약 6분

기존 `model Copy { ... }` 블록을 아래로 통째로 바꿔주세요.

```prisma
// 실물 개별 자료 (소장 부수) — 실물이 있는 모든 종류에 등록
model Copy {
  id             Int        @id @default(autoincrement())

  libraryId      Int
  library        Library    @relation(fields: [libraryId], references: [id])
  materialId     Int
  material       Material   @relation(fields: [materialId], references: [id])

  registrationNo String     // 등록번호(고유번호) — 도서관 내 고유
  callNumber     String?    // 청구기호 (이용자가 자료 찾는 번호, 시리즈·COPY 번호 포함)
  specialCode    String?    // 별치기호
  shelfNo        String?    // 서가번호
  location       String?    // 소장처 (예: 2층 문학)
  memo           String?    // 메모
  status         CopyStatus @default(AVAILABLE)

  createdById    Int?       // 이 실물을 등록한 사서
  createdBy      User?      @relation(fields: [createdById], references: [id])

  createdAt      DateTime   @default(now())
  updatedAt      DateTime   @updatedAt

  @@unique([libraryId, registrationNo])
  @@index([materialId])
}
```

맨 아래 `CopyStatus` enum은 그대로 두세요. 저장합니다.

## 5. 자료 종류 목록 만들기 — 약 4분

(이전 버전에서 이미 `material-types.ts`를 만드셨다면 이 단계는 건너뛰세요.)

`frontend` → `src` → `lib` 폴더에 **`material-types.ts`** 파일을 만들어 붙여넣습니다.

```ts
// 자료 종류 목록.
// usesMarc: 책·DVD처럼 KOLIS-NET MARC로 등록하는 종류인지 여부.
export const MATERIAL_TYPES = [
  { code: "book",       usesMarc: true  }, // 도서
  { code: "dvd",        usesMarc: true  }, // DVD
  { code: "article",    usesMarc: false }, // 기사
  { code: "thesis",     usesMarc: false }, // 논문
  { code: "law",        usesMarc: false }, // 법령
  { code: "video",      usesMarc: false }, // 영상
  { code: "music",      usesMarc: false }, // 음악
  { code: "webpage",    usesMarc: false }, // 웹페이지
  { code: "boardgame",  usesMarc: false }, // 보드게임
  { code: "tool",       usesMarc: false }, // 공구
  { code: "equipment",  usesMarc: false }, // 장비
  { code: "collection", usesMarc: false }, // 자료집
  { code: "photo",      usesMarc: false }, // 사진
  { code: "clipping",   usesMarc: false }, // 기사스크랩
  { code: "etc",        usesMarc: false }, // 기타
] as const;

export type MaterialTypeCode = (typeof MATERIAL_TYPES)[number]["code"];
export const MATERIAL_TYPE_CODES = MATERIAL_TYPES.map((m) => m.code);
```
