# 7단계 (2) — 회원가입 기능 만들기

> 목표: 홈페이지에 회원가입 화면을 만들어, 실제 회원을 사용자 표에 저장합니다.
> 이때 비밀번호를 안전하게 "암호화"해서 저장하는 걸 배웁니다.
> 소요 시간: 약 45분

---

## 오늘의 위치

7단계(계정 시스템) 네 부분 중 **두 번째**입니다.

1. 사용자 표 만들기 ✅
2. **회원가입 기능 ← 오늘**
3. 로그인 기능
4. 관리자 페이지 자물쇠

오늘 배우는 가장 중요한 개념은 **비밀번호 암호화**예요. 회원이 입력한 비밀번호를 **그대로 저장하면 절대 안 됩니다.** 만에 하나 데이터베이스가 유출되면 모든 회원의 비밀번호가 그대로 새어나가니까요. 그래서 비밀번호는 되돌릴 수 없게 뒤섞어서(암호화해서) 저장합니다. 이 일을 해주는 도구가 bcrypt예요.

작업은 두 덩어리입니다. **주방 쪽**에 "가입 처리 창구"를 만들고, **홀 쪽**에 가입 화면을 만듭니다.

---

## 준비

주방·홀 두 서버를 켜두세요.
- 터미널 1: backend에서 `npm run start:dev`
- 터미널 2: frontend에서 `npm run dev`

주방 서버가 도는 터미널 말고 **다른 터미널**에서 backend 폴더로 가 설치합니다.

```

```

- `bcryptjs`가 비밀번호를 암호화해주는 도구예요. (`@3`은 버전을 3으로 지정 — 이 버전은 타입 정보가 들어 있어 별도 설정이 필요 없습니다.)

## A-2. 가입 처리 일꾼 만들기 — 약 10분

`backend` → `src` 안에 **`users`** 폴더를 새로 만들고, 그 안에 **`users.service.ts`** 파일을 만들어 아래를 붙여넣습니다.

```typescript
import {
  Injectable,
  ConflictException,
  BadRequestException,
} from '@nestjs/common';
import { PrismaService } from '../prisma.service';
import * as bcrypt from 'bcryptjs';
import { randomUUID } from 'crypto';

// 가입 화면에서 넘어오는 정보의 모양
type SignupData = {
  loginId: string;
  password: string;
  name: string;
  phone: string;
  email?: string;
};

@Injectable()
export class UsersService {
  constructor(private prisma: PrismaService) {}

  async signup(data: SignupData) {
    // 1. 필수 값이 다 들어왔는지 확인
    if (!data.loginId || !data.password || !data.name || !data.phone) {
      throw new BadRequestException(
        '아이디, 비밀번호, 이름, 휴대폰 번호는 필수입니다.',
      );
    }

    // 2. 어느 도서관 소속인지 (지금은 첫 번째 도서관. 나중에 도메인으로 판별)
    const library = await this.prisma.library.findFirst();
    if (!library) {
      throw new BadRequestException('도서관 설정이 없습니다.');
    }

    // 3. 같은 도서관에 같은 아이디가 이미 있는지 확인
    const existing = await this.prisma.user.findFirst({
      where: { libraryId: library.id, loginId: data.loginId },
    });
    if (existing) {
      throw new ConflictException('이미 사용 중인 아이디입니다.');
    }

    // 4. 비밀번호 암호화 (되돌릴 수 없게 뒤섞기)
    const passwordHash = await bcrypt.hash(data.password, 10);

    // 5. 회원증 보안 코드 자동 생성 (무작위 고유값, 나중에 QR용)
    const cardToken = randomUUID();

    // 6. 회원을 표에 저장 (역할=MEMBER, 상태=ACTIVE는 자동 기본값)
    await this.prisma.user.create({
      data: {
        libraryId: library.id,
        loginId: data.loginId,
        passwordHash,
        name: data.name,
        phone: data.phone,
        email: data.email || undefined,
        cardToken,
      },
    });

    // 7. 비밀번호는 빼고 안전한 정보만 돌려줍니다.
    return { message: '회원가입이 완료되었습니다.' };
  }
}
```

핵심 두 줄만 짚을게요. `bcrypt.hash(data.password, 10)` 이 비밀번호를 암호화합니다(뒤의 `10`은 "얼마나 꼼꼼히 뒤섞을지" 정도예요). `randomUUID()` 는 아무도 못 맞추는 무작위 회원증 코드를 하나 만들어 줍니다.

## A-3. 가입 요청을 받는 창구 만들기 — 약 5분

같은 `users` 폴더에 **`users.controller.ts`** 파일을 만들어 붙여넣습니다.

```typescript
import { Controller, Post, Body } from '@nestjs/common';
import { UsersService } from './users.service';

@Controller('users')
export class UsersController {
  constructor(private usersService: UsersService) {}

  // POST 요청: 새 회원 만들기(가입)
  @Post()
  signup(
    @Body()
    body: {
      loginId: string;
      password: string;
      name: string;
      phone: string;
      email?: string;
    },
  ) {
    return this.usersService.signup(body);
  }
}
```

> `@Post`는 "새로 만들기" 요청이에요. 앞에서 배운 `@Get`(가져오기), `@Patch`(바꾸기)와 짝을 이룹니다.

# B. 홀(Next.js) 쪽 작업 — 회원가입 화면

## B-1. /signup 페이지 만들기 — 약 15분

`frontend` → `src` → `app` 폴더 안에 **`signup`** 폴더를 만들고, 그 안에 **`page.tsx`** 파일을 만들어 아래를 붙여넣습니다. (5·6단계에서 쓴 shadcn 부품을 그대로 활용해요.)

```tsx
"use client";

import { useState } from "react";
import { Button } from "@/components/ui/button";
import { Input } from "@/components/ui/input";
import { Label } from "@/components/ui/label";
import { Card, CardContent, CardHeader, CardTitle } from "@/components/ui/card";

const API_URL = "http://localhost:3001";

export default function SignupPage() {
  const [loginId, setLoginId] = useState("");
  const [password, setPassword] = useState("");
  const [name, setName] = useState("");
  const [phone, setPhone] = useState("");
  const [email, setEmail] = useState("");
  const [status, setStatus] = useState("");

  async function handleSignup() {
    setStatus("가입 중...");
    const res = await fetch(`${API_URL}/users`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ loginId, password, name, phone, email }),
    });
    if (res.ok) {
      setStatus("✅ 회원가입이 완료되었습니다!");
    } else {
      const data = await res.json().catch(() => null);
      setStatus("❌ " + (data?.message || "가입에 실패했습니다."));
    }
  }

  return (
    <main className="mx-auto max-w-md p-8">
      <Card>
        <CardHeader>
          <CardTitle>회원가입</CardTitle>
        </CardHeader>
        <CardContent className="flex flex-col gap-4">
          <div className="flex flex-col gap-2">
            <Label htmlFor="loginId">아이디</Label>
            <Input id="loginId" value={loginId} onChange={(e) => setLoginId(e.target.value)} />
          </div>
          <div className="flex flex-col gap-2">
            <Label htmlFor="password">비밀번호</Label>
            <Input id="password" type="password" value={password} onChange={(e) => setPassword(e.target.value)} />
          </div>
          <div className="flex flex-col gap-2">
            <Label htmlFor="name">이름</Label>
            <Input id="name" value={name} onChange={(e) => setName(e.target.value)} />
          </div>
          <div className="flex flex-col gap-2">
            <Label htmlFor="phone">휴대폰 번호</Label>
            <Input id="phone" value={phone} onChange={(e) => setPhone(e.target.value)} placeholder="010-1234-5678" />
          </div>
          <div className="flex flex-col gap-2">
            <Label htmlFor="email">이메일 (선택)</Label>
            <Input id="email" type="email" value={email} onChange={(e) => setEmail(e.target.value)} />
          </div>
          <Button onClick={handleSignup}>가입하기</Button>
          <p className="text-sm text-muted-foreground">{status}</p>
        </CardContent>
      </Card>
    </main>
  );
}
```

> 비밀번호 입력창은 `type="password"` 라서 입력할 때 점(●●●)으로 가려집니다.
> 생년월일·주소·우편번호는 지금은 화면에서 뺐어요. 나중에 같은 방식으로 쉽게 추가할 수 있습니다.

## C. 확인하기 (오늘의 하이라이트: 암호화 눈으로 보기!) — 약 5분

1. 브라우저에서 `http://localhost:3000/signup` 접속
2. 아이디·비밀번호·이름·휴대폰 번호를 채우고 **가입하기** 클릭
3. "✅ 회원가입이 완료되었습니다!" 가 뜨면 성공.

이제 진짜 재미있는 부분이에요. 방금 저장된 회원을 들여다봅시다.

4. 새 터미널에서 `cd C:\projects\LibraryNearMe\backend` 후 `npx prisma studio` 실행
5. `User` 표를 클릭해 방금 가입한 회원 줄을 봅니다.

✅ **확인하기**: `passwordHash` 칸을 보세요. 여러분이 입력한 비밀번호가 **그대로 있지 않고**, `$2a$10$...` 로 시작하는 알아볼 수 없는 긴 문자열로 바뀌어 있을 거예요. 이게 바로 암호화입니다! 이렇게 저장하면 이 표를 누가 훔쳐봐도 원래 비밀번호는 알 수 없어요. 그리고 `cardToken` 칸에는 무작위 회원증 코드가, `role`에는 `MEMBER`가 자동으로 들어간 것도 확인해 보세요. 🎉

**한 가지 더 테스트:** 다시 `/signup`으로 가서 **똑같은 아이디로** 가입을 시도해 보세요. "❌ 이미 사용 중인 아이디입니다." 가 뜨면, 아이디 중복 방지도 잘 작동하는 거예요.

확인 후 Prisma Studio는 `Ctrl+C`로 꺼주세요.
