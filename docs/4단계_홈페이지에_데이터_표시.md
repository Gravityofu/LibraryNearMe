# 4단계 — 홈페이지에 내 도서관 띄우기

> 목표: 홀(Next.js)이 주방(NestJS)에게서 도서관 설정을 받아와,
> 홈페이지 화면에 도서관 이름과 색상을 실제로 표시합니다.
> 소요 시간: 약 30분

---

## 오늘의 그림

지금까지 주방(NestJS)은 `http://localhost:3001/library` 로 데이터를 내어줄 준비가 됐습니다. 오늘은 홀(Next.js)이 그 데이터를 받아다가 손님(방문자)에게 보여줍니다.

```
[손님의 브라우저]  ←보여줌←  [홀 Next.js]  ←데이터 받아옴←  [주방 NestJS]  ←  [창고 DB]
```

### 잠깐, CORS는 안 만나나요?

3단계 끝에서 "CORS라는 보안 규칙을 만날 것"이라고 했는데, 오늘은 안 만납니다. 이유가 재미있어요.

CORS는 **"손님의 브라우저"가 다른 곳(주방)에 직접 전화를 걸 때** 작동하는 보안 규칙입니다. 그런데 오늘 우리가 쓸 Next.js 방식은, **홀(Next.js 서버)이 손님을 대신해서 주방에 다녀오는** 구조예요. 서버끼리의 대화는 이 보안 규칙의 대상이 아니라서, CORS를 켤 필요가 없습니다. 이게 요즘 Next.js(App Router)의 기본이자 더 깔끔한 방식이에요.

(CORS는 나중에 관리자 페이지에서 손님 브라우저가 직접 데이터를 저장/수정할 때 만나게 됩니다. 그때 차근차근 다룰게요.)

## 1. 홈페이지 화면 코드 바꾸기 — 약 10분

지금 `localhost:3000`에 뜨는 화면은 Next.js가 기본으로 넣어둔 환영 페이지입니다. 이걸 우리 도서관 화면으로 바꿉니다.

VS Code에서 `frontend` → `src` → `app` → **`page.tsx`** 파일을 엽니다. 안에 있는 내용을 **전부 지우고**, 아래 내용을 통째로 붙여넣은 뒤 저장(`Ctrl+S`)하세요.

```tsx
// 이 파일은 홈페이지의 첫 화면입니다.
// 함수 앞에 async가 붙어 있어서, 화면을 그리기 전에
// 주방(NestJS)에서 도서관 데이터를 먼저 받아옵니다.

// 도서관 데이터의 모양을 미리 알려줍니다. (3단계에서 만든 표와 같은 모양)
type Library = {
  id: number;
  name: string;
  logoUrl: string | null;
  primaryColor: string;
};

// 주방의 주소입니다. 나중에 배포할 때 이 주소만 실제 서버 주소로 바꾸면 됩니다.
const API_URL = "http://localhost:3001";

// 주방에 가서 도서관 설정을 받아오는 함수입니다.
async function getLibrary(): Promise<Library | null> {
  const res = await fetch(`${API_URL}/library`, { cache: "no-store" });
  if (!res.ok) return null;
  return res.json();
}

// 실제 홈페이지 화면을 그리는 부분입니다.
export default async function Home() {
  const library = await getLibrary();

  // 도서관 데이터가 없을 때(또는 주방이 꺼졌을 때)를 대비한 안내 화면입니다.
  if (!library) {
    return (
      <main style={{ padding: "2rem", textAlign: "center" }}>
        도서관 설정을 불러오지 못했습니다. 주방(백엔드) 서버가 켜져 있는지 확인해 주세요.
      </main>
    );
  }

  return (
    <main
      style={{
        minHeight: "100vh",
        display: "flex",
        flexDirection: "column",
        alignItems: "center",
        justifyContent: "center",
        gap: "1rem",
        fontFamily: "sans-serif",
      }}
    >
      {/* 도서관 이름을, 데이터베이스에 저장된 대표 색상으로 표시합니다 */}
      <h1
        style={{
          fontSize: "3rem",
          fontWeight: "bold",
          color: library.primaryColor,
        }}
      >
        {library.name}
      </h1>
      <p style={{ color: "#666" }}>도서관 홈페이지에 오신 것을 환영합니다</p>
    </main>
  );
}
```

붙여넣은 코드에서 눈여겨볼 딱 두 줄:
- `const library = await getLibrary();` → "주방에서 데이터를 받아올 때까지 기다렸다가 진행해"
- `color: library.primaryColor` → 도서관 이름 색을 **데이터베이스에 저장된 색**으로 칠하기

## 2. 확인하기 — 브라우저에서 보기 — 약 5분

브라우저에서 `http://localhost:3000` 에 접속(또는 이미 열려 있으면 새로고침 `F5`)합니다.

✅ **확인하기**: 화면 한가운데에 **조제님이 3단계에서 넣은 도서관 이름**(예: "무지개 도서관")이 색깔 있는 큰 글씨로 뜨면 성공입니다! 🎉

이 이름과 색깔이 어디서 왔는지 떠올려 보세요. 조제님이 Prisma Studio로 창고에 넣은 데이터 → 주방이 꺼내서 → 홀이 받아다가 → 지금 화면에 그린 것입니다. 데이터가 실제로 화면까지 도달했어요!

> 만약 "불러오지 못했습니다" 안내가 뜨면, 주방(터미널 1)이 켜져 있는지 확인하세요. `localhost:3001/library`가 여전히 잘 나오는지 먼저 보고, 그다음 홈페이지를 새로고침하면 됩니다.

## 3. 진짜 마법 확인하기 (지침의 핵심 기능 미리보기!) — 약 5분

지침에 "도서관마다 이름·색상을 다르게 보이게 한다"는 내용이 있었죠? 그 기능의 씨앗이 이미 작동하는 걸 지금 눈으로 확인할 수 있습니다.

1. **새 터미널**을 하나 열고 Prisma Studio를 켭니다.
   ```
   cd C:\projects\LibraryNearMe\backend
   npx prisma studio
   ```
2. `Library` 표에서 아까 넣은 도서관 줄을 클릭해, `primaryColor` 값을 다른 색으로 바꿔보세요. 예: `#e11d48` (빨강) 또는 `#16a34a` (초록). `name`도 다른 이름으로 바꿔도 됩니다.
3. `Save` 로 저장합니다.
4. 이제 브라우저에서 홈페이지(`localhost:3000`)를 **새로고침(F5)** 합니다.

✅ **확인하기**: 도서관 이름의 **색깔(과 이름)이 방금 바꾼 값으로 변하면**, 축하합니다! 코드를 한 줄도 안 고치고 데이터만 바꿔서 화면을 바꾼 거예요. 이게 바로 여러 도서관이 각자의 이름·색상을 갖게 되는 시스템의 원리입니다.

확인했으면 Prisma Studio 터미널은 `Ctrl+C`로 꺼주세요.
