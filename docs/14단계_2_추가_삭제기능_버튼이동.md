## 1. 백엔드 — 삭제 API 추가 (약 5분)

### 1-1. 서비스에 삭제 함수 추가

`backend/src/settings/kormarc-tags.service.ts` 를 엽니다. `update` 함수 **바로 아래**에 아래 함수를 추가하세요.

```typescript
  async remove(libraryId: number, id: number) {
    const existing = await this.prisma.kormarcTag.findFirst({
      where: { id, libraryId },
    });
    if (!existing) {
      throw new NotFoundException('태그를 찾을 수 없습니다.');
    }
    await this.prisma.kormarcTag.delete({ where: { id } });
    return { success: true };
  }
```

- 다른 도서관의 태그를 실수로/악의적으로 지우지 못하도록, 먼저 `libraryId`로 이 도서관 소유가 맞는지 확인한 뒤에만 삭제합니다.

### 1-2. 컨트롤러에 삭제 경로(route) 추가

`backend/src/settings/kormarc-tags.controller.ts` 를 엽니다.

**① import 줄 수정** — 맨 위 import에 `Delete`를 추가하세요.

```typescript
import { Controller, Get, Post, Patch, Delete, Body, Param, Req, UseGuards } from '@nestjs/common';
```

**② 라우트 추가** — `update` 메서드 바로 아래에 추가하세요.

```typescript

```

이제 `DELETE /settings/kormarc-tags/:id` 로 태그를 지울 수 있어요. 저장하면 backend가 재시작됩니다.

## 3. `admin/settings/page.tsx` 수정 (약 8분)

### 3-1. 삭제 함수 추가

`handleSave` 함수 **바로 아래**에 아래 함수를 추가하세요.

```tsx
  async function handleDelete() {
    if (!editingId) return;
    if (!window.confirm(t("settings.tags.deleteConfirm"))) return;
    const token = localStorage.getItem("token");
    if (!token) return;
    const res = await fetch(`${API_URL}/settings/kormarc-tags/${editingId}`, {
      method: "DELETE",
      headers: { Authorization: `Bearer ${token}` },
    });
    if (res.ok) {
      notify("✅ " + t("settings.tags.deleteSuccess"), "success");
      setShowModal(false);
      await loadTags();
    } else {
      notify("❌ " + t("settings.tags.deleteFail"), "error");
    }
  }
```

- `window.confirm(...)`는 "정말 삭제하시겠습니까?" 같은 브라우저 기본 확인창을 띄워서, 실수로 삭제 버튼을 눌러도 한 번 더 물어봐줘요. "확인"을 눌러야만 삭제가 진행됩니다.
- 삭제는 지금 수정 중인 태그(`editingId`)를 대상으로 하므로, "새 태그 추가" 모달(아직 `editingId`가 없는 상태)에서는 이 함수가 호출돼도 아무 일도 안 일어나요. 다음 단계에서 버튼 자체를 그 상태일 때는 안 보이게 만들 거예요.

### 3-3. 모달 하단에 빨간 삭제 버튼 추가

아래 부분을 찾으세요. (모달 안, 저장 버튼이 있는 자리예요.)

```tsx
            <button
              onClick={handleSave}
              className="mt-5 w-full cursor-pointer rounded-lg bg-[#383838] py-2.5 text-sm font-semibold text-[#F9F6F0]"
            >
              {t("settings.tags.save")}
            </button>
          </div>
        </div>
      )}
```

이렇게 바꿔주세요. (저장 버튼은 그대로 두고, 수정 중일 때만 그 아래에 빨간 삭제 버튼을 추가했어요.)

```tsx
            <button
              onClick={handleSave}
              className="mt-5 w-full cursor-pointer rounded-lg bg-[#383838] py-2.5 text-sm font-semibold text-[#F9F6F0]"
            >
              {t("settings.tags.save")}
            </button>
            {editingId && (
              <button
                onClick={handleDelete}
                className="mt-2 w-full cursor-pointer rounded-lg bg-red-600 py-2.5 text-sm font-semibold text-white hover:bg-red-700"
              >
                {t("settings.tags.deleteBtn")}
              </button>
            )}
          </div>
        </div>
      )}
```

- `{editingId && (...)}` 는 "지금 수정 중인 태그가 있을 때만" 이라는 뜻이에요. "새 태그 추가" 모달을 열었을 땐 `editingId`가 `null`이라 삭제 버튼이 안 보이고, "수정" 버튼으로 연 모달에서만 보입니다.
- `bg-red-600`은 tailwind 기본 빨간색이에요. 마우스를 올리면 `hover:bg-red-700`로 살짝 더 진해집니다.
